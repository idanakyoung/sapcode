# 1. ZDEMO_01 : 데이터 Create / Update / Delete [ZGW_DEMO_G01]

## 1.1 프로젝트 개요

이 예제는 SAPUI5 + ODataModel 기반으로  
`InfoSet` 엔터티에 대해 **조회(Search) + 생성(Create) + 수정(Update) + 삭제(Delete)** 를 수행하는 실습이다.

핵심 구조는 아래와 같다.

```text
화면 Input / DatePicker
   ↓
Info(JSONModel)
   ↓
Controller에서 payload 생성
   ↓
ODataModel.create / update / remove
   ↓
Gateway Service (ZGW_DEMO_G01)
```

---

## 1.2 Main Controller

```javascript
sap.ui.define([
    "sap/ui/core/mvc/Controller"
], (Controller) => {
    "use strict";

    return Controller.extend("code.zdemo01.controller.Main", {
        onInit() {
        },

        onSearch() {
            let condCarrid = this.getView().byId("InpCarrid").getValue();

            let oModel = this.getView().getModel();
            let sPath = oModel.createKey("/InfoSet", {
                Carrid: condCarrid
            });

            //alert( sPath );

            let oView = this.getView();

            oModel.read(sPath, {
                success: function( oData, oResponse ) {
                    let jsonModel = new sap.ui.model.json.JSONModel({});
                    oView.setModel(jsonModel, "Info");
                    oView.getModel("Info").setProperty("/demo", oData);
                    oView.byId("PanelInfo").bindElement("Info>/demo");
                },
                error: function( oError ) {
                    let oMesg = JSON.parse(oError.responseText);
                    sap.m.MessageToast.show( oMesg.error.message.value );
                    let oPanelInfo = oView.byId("PanelInfo").unbindElement();
                    oPanelInfo.unbindElement();
                }
            });
        },
        onInsert: function () {
            let oModelInsert = this.getView().getModel("Info");
            let oDataInsert = oModelInsert.getData();
            // console.log(oDataInsert);

            let entity = {
                "Carrid" : oDataInsert.demo.Carrid,
                "Begda" : this._DateTime(oDataInsert.demo.Begda),
                "Endda" : this._DateTime(oDataInsert.demo.Endda),
                "Price" : new String(oDataInsert.demo.Price),
                "Waers" : oDataInsert.demo.Waers
            };

            let oModel = this.getView().getModel();
            let sPath = "/InfoSet";
            oModel.create(sPath, entity, {
                success: function(oData, oResponse) {
                    alert("Success, Create entity");
                },
                error: function(oError) {
                    alert("Error, Create entity");
                }
            });
        },
        _DateTime(oDate) {
            let oDateTimeFormat = sap.ui.core.format.DateFormat.getDateTimeInstance({
                pattern: "yyyy-MM-dd'T'HH:mm:ss",
                UTC: false  //Gateway Server에 사용자가 설정한 Local Time Zone 그대로 보냄
            });
            return oDateTimeFormat.format(oDate);
        },
        onUpdate: function() {
            let oModelUpdate = this.getView().getModel("Info");
            let oDataUpdate = oModelUpdate.getData();

            let oModel = this.getView().getModel();

            let sPath = oModel.createKey("/InfoSet", {
                Carrid: oDataUpdate.demo.Carrid
            });

            let oEntity = {
                Begda: this._DateTime(oDataUpdate.demo.Begda),
                Endda: this._DateTime(oDataUpdate.demo.Endda),
                Price: new String(oDataUpdate.demo.Price),
                Waers: oDataUpdate.demo.Waers
            };

            oModel.update(
                sPath,
                oEntity,
                {
                    success: function(oData, oResponse) {
                        alert('Success, update entity');
                    },
                    error: function(oError) {
                        alert('Error, update entity');
                    }
                }
            );
        },
        onDelete: function () {
            let oModelDel = this.getView().getModel("Info");
            let oDataDel = oModelDel.getData();

            let oModel = this.getView().getModel();

            let sPath = oModel.createKey("/InfoSet", {
                Carrid: oDataDel.demo.Carrid
            });

            oModel.remove(sPath, {
                success: function(oData, oResponse) {
                    alert('Success, update entity');
                },
                error: function(oError) {
                    alert('Error, update entity');
                }
            })
        }
    });
});
```

## 1.4 DatePicker의 `valueFormat` 변경 포인트

### 변경 내용

```xml
valueFormat="yyyy.MM.dd"
→
valueFormat="yyyy-MM-dd"
```

### 이유
`_DateTime()` 함수가 최종적으로 날짜를

```text
yyyy-MM-dd'T'HH:mm:ss
```

형태로 만들어 보내기 때문에,  
`valueFormat`도 같은 계열인 `yyyy-MM-dd`로 맞춰두는 것이 더 자연스럽다.

### 구분

| 속성 | 의미 |
|---|---|
| `displayFormat` | 화면에 보여주는 형식 (사용자용) |
| `valueFormat` | 모델에 실제 들어가는 값의 형식 (데이터용) |

---

## 1.5 `_DateTime(oDate)` 추가

### 왜 필요한가?

UI5 `DatePicker`가 주는 값은 보통 `Date 객체`이거나 날짜 타입 값이다.  
하지만 OData / Gateway는 날짜/시간을 특정 **문자열 형식**으로 받기를 기대하는 경우가 많다.

그래서 다음 변환이 필요하다.

```text
Date 객체
   ↓
"yyyy-MM-dd'T'HH:mm:ss"
```

### 코드

```javascript
_DateTime(oDate) {
    let oDateTimeFormat = sap.ui.core.format.DateFormat.getDateTimeInstance({
        pattern: "yyyy-MM-dd'T'HH:mm:ss",
        UTC: false
    });
    return oDateTimeFormat.format(oDate);
}
```

### 핵심 포인트
- `UTC: false`
  - Gateway Server에 사용자가 설정한 Local Time Zone 기준으로 보낸다는 의미
- 날짜가 하루 밀리는 문제를 줄이기 위해 UTC 옵션을 명확히 지정한 것

## 1.6 Search 기능 (`onSearch`)

### 목적
입력한 `Carrid`를 Key로 사용해서  
OData `InfoSet`에서 1건 조회한 뒤,  
그 결과를 `Info` JSONModel에 담아 화면에 바인딩한다.

### 흐름

```text
InpCarrid 입력값 읽기
   ↓
ODataModel.createKey("/InfoSet", { Carrid })
   ↓
oModel.read()
   ↓
성공 시 JSONModel("Info") 생성
   ↓
/demo 에 조회 결과 저장
   ↓
PanelInfo.bindElement("Info>/demo")
```

### 핵심 포인트
- `createKey()`를 써서 OData Key URL을 안전하게 생성
- `PanelInfo` 전체를 `bindElement`로 묶어서 내부 필드를 한 번에 바인딩

## 1.7 Insert 기능 (`onInsert`)

### 목적
화면에서 입력/조회한 데이터를 읽어  
OData Create 요청으로 새 엔터티를 생성한다.

### 핵심 코드

```javascript
let entity = {
    "Carrid" : oDataInsert.demo.Carrid,
    "Begda" : this._DateTime(oDataInsert.demo.Begda),
    "Endda" : this._DateTime(oDataInsert.demo.Endda),
    "Price" : new String(oDataInsert.demo.Price),
    "Waers" : oDataInsert.demo.Waers
};

oModel.create("/InfoSet", entity, { success, error });
```

### 해석

#### 1) JSONModel에서 값 가져오기
```javascript
let oModelInsert = this.getView().getModel("Info");
let oDataInsert = oModelInsert.getData();
```

#### 2) 서버로 보낼 payload 생성
- `Begda`, `Endda`
  - `_DateTime()`으로 날짜 문자열 변환
- `Price`
  - 문자열로 변환해서 안정적으로 전달

#### 3) OData Create 호출
```javascript
oModel.create("/InfoSet", entity, ...)
```

### 흐름

```text
Info 모델 값 읽기
   ↓
entity 생성
   ↓
Date → 문자열 변환
   ↓
Price → String 변환
   ↓
oModel.create("/InfoSet", entity)
```

---

## 1.8 Update 기능 (`onUpdate`)

### 목적
기존 데이터를 `Carrid` 키 기준으로 찾아  
일부 필드를 수정한다.

### 핵심 코드

```javascript
let sPath = oModel.createKey("/InfoSet", {
    Carrid: oDataUpdate.demo.Carrid
});

let oEntity = {
    Begda: this._DateTime(oDataUpdate.demo.Begda),
    Endda: this._DateTime(oDataUpdate.demo.Endda),
    Price: new String(oDataUpdate.demo.Price),
    Waers: oDataUpdate.demo.Waers
};

oModel.update(sPath, oEntity, { success, error });
```

### 해석

#### 1) Update는 대상 1건 지정이 중요
- `createKey("/InfoSet", { Carrid: ... })`
- 실제 경로 예시:
```text
/InfoSet(Carrid='AA')
```

#### 2) payload에는 수정할 값만 넣음
- `Carrid`는 URL Key에 이미 포함됨
- 보통 payload에는 변경 필드만 넣으면 됨

### 흐름

```text
Info 모델 값 읽기
   ↓
Carrid로 key path 생성
   ↓
수정할 필드만 oEntity에 담기
   ↓
oModel.update(sPath, oEntity)
```

---

## 1.9 Delete 기능 (`onDelete`)

### 목적
`Carrid` 키 기준으로 대상 엔터티를 삭제한다.

### 핵심 코드

```javascript
let sPath = oModel.createKey("/InfoSet", {
    Carrid: oDataDel.demo.Carrid
});

oModel.remove(sPath, {
    success: function(oData, oResponse) {
        alert('Success, update entity');
    },
    error: function(oError) {
        alert('Error, update entity');
    }
})
```

### 해석
- Delete도 Update처럼 **정확한 대상 경로**가 필요하다.
- `remove()`는 payload 없이 경로만으로 삭제한다.

### 흐름

```text
Info 모델 값 읽기
   ↓
Carrid로 key path 생성
   ↓
oModel.remove(sPath)
```

## 1.10 CRUD 비교표

| 기능 | 메서드 | 호출 형태 | 대상 지정 방식 | payload |
|---|---|---|---|---|
| Create | `oModel.create` | `create("/InfoSet", entity)` | 엔터티셋 경로 | 생성할 필드 필요 |
| Update | `oModel.update` | `update(sPath, oEntity)` | `createKey()`로 만든 key 경로 | 수정할 필드만 |
| Delete | `oModel.remove` | `remove(sPath)` | `createKey()`로 만든 key 경로 | 없음 |

# 2. ZDEMO_02_G01 : 테이블 컬럼 클릭 시 데이터 바인딩 [ZGW_NAVI_G01]

## 2.1 프로젝트 개요

이 코드는 SAPUI5에서 **Master-Detail 구조**를 구현하는 예제이며,  
OData의 **Association / Navigation** 개념을 UI에서 실습하는 내용이다.

### 구조 설명

| 영역 | 역할 |
|---|---|
| `tabCarrier` | 항공사 목록 (Master) |
| `tabConnect` | 선택한 항공사의 연결편 목록 (Detail) |

### 동작 흐름

```text
Carrier 1건 선택
   ↓
해당 Carrier의 to_Connect 네비게이션 데이터 표시
```

즉,  
**Carrier (1) : Connect (N)** 관계를 UI에서 표현하는 예제다.

---

## 2.2 Main Controller

```javascript
sap.ui.define([
    "sap/ui/core/mvc/Controller"
], (Controller) => {
    "use strict";

    return Controller.extend("code.zdemo02g01.controller.Main", {
        onInit() {
        },
        onSelectChange: function(oEvent) {
            let oSelectItem = oEvent.getParameter("listItem");
            let oContext = oSelectItem.getBindingContext();
            let sPath = oContext.getPath();
            alert(sPath);

            let oTable = this.getView().byId("tabConnect");
            oTable.bindElement(sPath);
        }
    });
});
```

## 2.4 첫 번째 테이블 `tabCarrier`

### 코드 핵심

```xml
<Table id="tabCarrier"
       items="{/CarrierSet}"
       growing="true"
       growingThreshold="10"
       selectionChange="onSelectChange"
       mode="SingleSelectMaster">
```

### 의미

- `items="{/CarrierSet}"`
  - ODataModel의 `CarrierSet` 전체를 리스트 바인딩
- `mode="SingleSelectMaster"`
  - 한 행만 선택 가능
- `selectionChange="onSelectChange"`
  - 선택이 바뀌면 Controller 이벤트 발생
- `growing="true"`
  - 많은 데이터일 때 점진 로딩

### 컬럼 의미

| 필드 | 의미 |
|---|---|
| `Carrid` | 항공사 코드 |
| `Carrname` | 항공사 이름 |
| `Currcode` | 통화 코드 |

---

## 2.5 두 번째 테이블 `tabConnect`

### 코드 핵심

```xml
<Table id="tabConnect" items="{to_Connect}">
```

### 가장 중요한 포인트
- 루트 경로(`/`)가 아니다
- 독립적인 엔터티셋이 아니다
- **현재 바인딩된 Carrier의 Navigation Property** 이다

즉,

```text
tabConnect는 혼자 동작하지 않음
   ↓
먼저 bindElement(sPath) 필요
   ↓
그 다음 items="{to_Connect}"가 의미를 가짐
```

---

## 2.6 Controller의 `onSelectChange` 해석

### 코드

```javascript
onSelectChange: function(oEvent) {
    let oSelectItem = oEvent.getParameter("listItem");
    let oContext = oSelectItem.getBindingContext();
    let sPath = oContext.getPath();
    alert(sPath);

    let oTable = this.getView().byId("tabConnect");
    oTable.bindElement(sPath);
}
```

### 동작 단계

1. 사용자가 Carrier 행 선택
2. 선택된 행의 Binding Context 가져오기
3. 해당 행의 OData 경로 추출
4. `tabConnect`에 `bindElement(sPath)` 적용

### 코드 의미 분석

#### 1) 선택된 행 객체 가져오기
```javascript
let oSelectItem = oEvent.getParameter("listItem");
```

#### 2) 해당 행의 Binding Context 가져오기
```javascript
let oContext = oSelectItem.getBindingContext();
```

예시 경로:
```text
/CarrierSet('AA')
```

#### 3) 실제 경로 문자열 추출
```javascript
let sPath = oContext.getPath();
```

#### 4) Detail Table에 기준 경로 설정
```javascript
oTable.bindElement(sPath);
```

## 2.7 Navigation Binding이 실제로 일어나는 방식

`tabConnect`에는

```xml
items="{to_Connect}"
```

가 걸려 있다.

그리고 Controller에서

```javascript
oTable.bindElement("/CarrierSet('AA')");
```

를 수행하면,

실제 내부 요청은 아래처럼 된다.

```text
/CarrierSet('AA')/to_Connect
```

이것이 **Navigation Binding** 이다.

---

## 2.8 핵심 개념 비교표

| 개념 | 코드 | 의미 |
|---|---|---|
| List Binding | `items="{/CarrierSet}"` | 엔터티셋 전체 조회 |
| Element Binding | `bindElement(sPath)` | 특정 1건 기준 바인딩 |
| Navigation Binding | `items="{to_Connect}"` | 현재 엔터티의 연관 데이터 조회 |
| Association | Carrier 1 : N Connect | 1:N 관계 구조 |

# 3. PROJECT_01 : Create / Change / Cancel [ZGW_NAVI_G01]

## 3.1 프로젝트 개요

이 코드는 SAPUI5 + ODataModel 기반으로  
Carrier 데이터를 대상으로 아래 3가지를 실습한 예제다.

- Create (생성) → `oModel.create`
- Update (수정) → `oModel.update`
- Delete (삭제) → `oModel.remove`
  - 코드/버튼 이름은 `Cancel`

화면은 `SimpleForm` 안에 입력 필드 3개와 버튼 3개로 구성된다.

## 3.2 Main Controller

```javascript
sap.ui.define([
    "sap/ui/core/mvc/Controller"
], (Controller) => {
    "use strict";

    return Controller.extend("code.project01.controller.Main", {
        onInit() {
            let jsonModel = new sap.ui.model.json.JSONModel({});
            this.getView().setModel(jsonModel, "Info");
        },

        onCreate() {
            let oModelCreate = this.getView().getModel("Info");
            let oDataCreate = oModelCreate.getData();
            // console.log(oDataCreate);

            let entity = {
                "Carrid" : oDataCreate.Carrid,
                "Carrname" : oDataCreate.Carrname,
                "Currcode" : oDataCreate.Currcode
            };

            let oModel = this.getView().getModel();
            let sPath = "/CarrierSet";
            oModel.create(sPath, entity, {
                success: function(oData, oResponse) {
                    alert("Success, Create entity");
                },
                error: function(oError) {
                    alert("Error, Create entity");
                }
            })
        },

        onChange: function() {
            let oModelChange = this.getView().getModel("Info");
            let oDataChange = oModelChange.getData();

            let oModel = this.getView().getModel();

            let sPath = oModel.createKey("/CarrierSet", {
                Carrid: oDataChange.Carrid
            });

            let oEntity = {
                Carrname: oDataChange.Carrname,
                Currcode: oDataChange.Currcode
            };

            oModel.update(
                sPath,
                oEntity,
                {
                    success: function(oData, oResponse) {
                    alert("Success, Change entity");
                    },
                    error: function(oError) {
                        alert("Error, Change entity");
                    }
                }
            );

            // console.log(oDataChange);
            // console.log(oDataChange.Carrid);
        },

        onCancel: function() {
            let oModelCan = this.getView().getModel("Info");
            let oDataCan = oModelCan.getData();

            let oModel = this.getView().getModel();

            let sPath = oModel.createKey("/CarrierSet", {
                Carrid: oDataCan.Carrid
            });

            oModel.remove(sPath, {
                success: function(oData, oResponse) {
                    alert('Success, cancel entity');
                },
                error: function(oError) {
                    alert('Error, cancel entity');
                }
            })
        }
    });
});
```

## 3.4 화면 구조 설명

| UI 영역 | 컨트롤 | 역할 |
|---|---|---|
| 버튼 영역 | `HBox + Button 3개` | Create / Change / Cancel 실행 |
| 입력 영역 | `Label + Input 3세트` | Carrid, Carrname, Currcode 입력/표시 |

### SimpleForm 특징
- Label/Input을 폼 형태로 정리해줌

## 3.5 버튼 영역 의미

| 버튼 | press 이벤트 | OData 동작 | 의미 |
|---|---|---|---|
| Create | `onCreate` | `create` | 신규 항공사 생성 |
| Change | `onChange` | `update` | 기존 항공사 수정 |

### UI 속성 포인트
- `icon`
  - SAP 기본 아이콘 사용
- `type`
  - `Accept / Emphasized / Reject`
  - 버튼 의미를 색과 강조로 표현
- `class="sapUiSmallMarginBegin"`
  - 버튼 간 간격 추가

## 3.6 입력 필드 바인딩 구조

### 코드

```xml
<Input id="InpCarrid" value="{Info>/Carrid}"/>
<Input id="InpCarrname" value="{Info>/Carrname}"/>
<Input id="InpCurrcode" value="{Info>/Currcode}"/>
```

### 핵심 포인트
- 기본 모델(ODataModel)이 아니라
- 별도의 JSONModel `"Info"`를 만들어 입력값을 담는다.
- 사용자가 입력한 값은 `Info` 모델에 저장되고
- CRUD 시 payload로 사용된다.

### 필드 의미

| 필드 | 바인딩 | 의미 |
|---|---|---|
| `Carrid` | `Info>/Carrid` | Key 값, 수정/삭제 대상 지정 |
| `Carrname` | `Info>/Carrname` | 항공사 이름 |
| `Currcode` | `Info>/Currcode` | 통화 코드 |

## 3.7 `onInit` 초기 설정

### 코드

```javascript
onInit() {
    let jsonModel = new sap.ui.model.json.JSONModel({});
    this.getView().setModel(jsonModel, "Info");
}
```

### 의미
- 화면이 처음 뜰 때 `Info` JSONModel을 빈 객체로 생성
- Input의 `value="{Info>/...}"` 바인딩이 동작할 수 있게 기반 마련
- 사용자가 입력한 값이 자동으로 Info 모델에 반영됨
  - Two-way binding

## 3.8 Create 기능 (`onCreate`)

### 목표
입력한 `Carrid / Carrname / Currcode`를 읽어  
`/CarrierSet`에 신규 엔터티를 생성한다.

### 핵심 코드

```javascript
let entity = {
    "Carrid" : oDataCreate.Carrid,
    "Carrname" : oDataCreate.Carrname,
    "Currcode" : oDataCreate.Currcode
};

oModel.create("/CarrierSet", entity, { success, error });
```

### 핵심 포인트

#### 1) Info 모델에서 입력값 가져오기
```javascript
let oModelCreate = this.getView().getModel("Info");
let oDataCreate = oModelCreate.getData();
```

#### 2) payload(entity) 생성
- Create는 보통 Key 포함해서 보내는 경우가 많다.

#### 3) ODataModel로 create 수행
```javascript
oModel.create("/CarrierSet", entity, ...)
```

### 흐름

```text
Info 모델 값 읽기
   ↓
entity 생성
   ↓
/CarrierSet 로 create 호출
   ↓
성공/실패 메시지 표시
```

---

## 3.9 Update 기능 (`onChange`)

### 목표
`Carrid`로 기존 엔터티를 찾고  
`Carrname`, `Currcode`를 수정한다.

### 핵심 코드

```javascript
let sPath = oModel.createKey("/CarrierSet", {
    Carrid: oDataChange.Carrid
});

let oEntity = {
    Carrname: oDataChange.Carrname,
    Currcode: oDataChange.Currcode
};

oModel.update(sPath, oEntity, { success, error });
```

### 핵심 포인트

#### 1) 대상 경로 생성
- `createKey("/CarrierSet", { Carrid })`
- 예:
```text
/CarrierSet('AA')
```

#### 2) payload에는 수정할 필드만
- `Carrid`는 경로에 이미 있으므로 보통 payload에서 제외 가능

#### 3) update 수행
```javascript
oModel.update(sPath, oEntity, ...)
```

### 흐름

```text
Info 모델 값 읽기
   ↓
Carrid로 key path 생성
   ↓
수정 필드만 oEntity 구성
   ↓
update 수행
```

---

## 3.10 Delete 기능 (`onCancel`)

### 목표
`Carrid` 키를 기준으로 항공사를 삭제한다.

### 핵심 코드

```javascript
let sPath = oModel.createKey("/CarrierSet", {
    Carrid: oDataCan.Carrid
});

oModel.remove(sPath, { success, error });
```

### 핵심 포인트
- Delete도 Update처럼 정확한 대상 경로가 필요
- `remove()`는 payload 없이 key path만 사용

### 흐름

```text
Info 모델 값 읽기
   ↓
Carrid로 key path 생성
   ↓
remove 수행
```

---

## 3.11 CRUD 메서드 비교표

| 기능 | 메서드 | 호출 형태 | 대상 지정 방식 | payload |
|---|---|---|---|---|
| Create | `oModel.create` | `create("/CarrierSet", entity)` | 엔터티셋 경로 | 생성할 필드 필요 |
| Update | `oModel.update` | `update(sPath, oEntity)` | `createKey()` key 경로 | 수정할 필드만 |
| Delete | `oModel.remove` | `remove(sPath)` | `createKey()` key 경로 | 없음 |

## 3.12 모델 역할 비교

| 모델 | 이름 | 역할 |
|---|---|---|
| JSONModel | `"Info"` | 화면 입력값 저장용 |
| ODataModel | 기본 모델 | 서버 CRUD 실제 수행 |

# 4. 전체 핵심 요약

## 4.1 ZDEMO_01
- Search로 OData 1건을 읽고
- JSONModel `"Info"`에 담아 화면에 바인딩
- Insert / Update / Delete는 모두 `"Info"` 모델 값을 기반으로 payload 생성

## 4.2 날짜 처리
- DatePicker 값은 그대로 보내지 말고
- `_DateTime()`으로 OData/Gateway가 기대하는 문자열 형식으로 변환

## 4.3 ZDEMO_02_G01
- Master-Detail 구조 예제
- `CarrierSet` 선택 시
- `to_Connect` 네비게이션 데이터를 `tabConnect`에 바인딩
- 핵심은 `bindElement(sPath)` + `items="{to_Connect}"`

## 4.4 PROJECT_01
- 입력폼 기반 CRUD 예제
- `Info` JSONModel은 화면 입력값 저장용
- 실제 서버 작업은 ODataModel이 수행
- Create는 엔터티셋 경로, Update/Delete는 key 경로 사용

## 4.5 가장 중요한 한 줄
> **UI5에서 CRUD와 Navigation Binding의 핵심은 “화면 입력/선택 상태를 별도 모델(JSONModel 또는 Binding Context)로 관리하고, 그 상태를 ODataModel 요청으로 정확히 연결하는 것”이다.**
