# 방법 2 **클래스 + IF_OO_ADT_CLASSRUN** ⭐

```csharp
/lrn/depment_rel (소스, 이미 데이터 있음)
        ↓*SELECT*
ZCL_..._CLASSRUN (클래스, 데이터 적재 로직)
        ↓*INSERT/ MODIFY*
zg01depment  (타겟 테이블, 실제 저장)
        ↓*SELECT*
zg01_r_department (CDSView, 조회/Association)
```

- 코드 예시 (**클래스** 내 메소드 코드)
    
    ```nasm
    METHOD if_oo_adt_classrun~main.
        
          DATA: lt_src TYPE TABLE OF /lrn/depment_rel,
                lt_tgt TYPE TABLE OF zg01depment,
                ls_tgt TYPE zg01depment.
        
          SELECT *
            FROM /lrn/depment_rel
            INTO TABLE @lt_src.
        
          out->write( |SRC rows (/lrn/depment_rel): { lines( lt_src ) }| ).  
        
          LOOP AT lt_src ASSIGNING FIELD-SYMBOL(<s>).
            CLEAR ls_tgt.
        
            ls_tgt-client               = sy-mandt.
            ls_tgt-id                   = <s>-id.
            ls_tgt-description          = <s>-description.
            ls_tgt-department_head      = <s>-head_id.
            ls_tgt-department_assistant = <s>-assistant_id.
        
            ls_tgt-created_by            = <s>-created_by.
            ls_tgt-created_at            = <s>-created_at.
            ls_tgt-local_last_changed_by = <s>-local_last_changed_by.
            ls_tgt-local_last_changed_at = <s>-local_last_changed_at.
            ls_tgt-last_changed_at       = <s>-last_changed_at.
        
            APPEND ls_tgt TO lt_tgt.
          ENDLOOP.
        
          TRY.
              MODIFY zg01depment FROM TABLE @lt_tgt.
              out->write( |DBCNT (written to zg01depment): { sy-dbcnt }| ).  
              COMMIT WORK.
              out->write( |COMMIT OK, SUBRC: { sy-subrc }| ).
            CATCH cx_sy_open_sql_db INTO DATA(lx).
              out->write( |DB ERROR: { lx->get_text( ) }| ).
          ENDTRY.
        
        ENDMETHOD.
    ```

# Implementing SQL Expressions in CDS Views Pushdown 이론

### 1. Code Pushdown 개념

- **계산·로직을 ABAP이 아니라 DB(HANA)에서 수행**
- ABAP은 **결과만 조회**
- 계산 위치를 **ABAP SQL → CDS View**로 이동

---

### 2. 왜 CDS로 Pushdown 하는가?

| 항목 | 설명 |
| --- | --- |
| 성능 | 대량 데이터 계산을 DB에서 처리 |
| 가독성 | ABAP SELECT 문이 단순해짐 |
| 재사용 | 여러 프로그램/서비스에서 동일 CDS 사용 |
| 일관성 | 계산 로직을 한 곳에서 관리 |

---

### 3. ABAP SQL vs CDS View (역할 비교)

| 구분 | ABAP SQL | CDS View |
| --- | --- | --- |
| 계산 정의 | 프로그램 내부 | CDS View 정의 |
| 계산 위치 | 애플리케이션 서버 | 데이터베이스 |
| 소비 방식 | 개별 SELECT | 공통 데이터 모델 |

---

### 4. Code Pushdown 예시

**ABAP SQL**

```abap
SELECT carrier_id,
       seats_max - seats_occupied AS seats_free
  FROM /dmo/flight
  INTO TABLE @DATA(result).
```

**CDS View**

```abap
define view entity Z_Sql_Logic
  as select from /dmo/flight {
    key carrier_id,
    seats_max - seats_occupied as seats_free
}
```

→ ABAP은 `seats_free`를 **일반 필드처럼 사용**

---

### 5. CDS View에 SQL 로직을 두는 장점

| 항목 | 의미 |
| --- | --- |
| Encapsulation | 계산 로직이 CDS에 숨겨짐 |
| Reuse | 분석, OData, BW 등에서 재사용 |
| Semantics | 의미 있는 필드명과 어노테이션 부여 |
| Authorization | CDS Access Control로 권한 제어 |

---

### 6. Literals in ABAP CDS

| 구분 | ABAP SQL | ABAP CDS |
| --- | --- | --- |
| 문자 리터럴 | CHAR | CHAR, NUMC |
| 정수 리터럴 | INT4 | INT1 / INT2 / INT4 자동 선택 |
| 실수 리터럴 | 지원 안 함 | FLTP 지원 |

---

### 7. CAST (Type Conversion)

```abap
cast( '19891109' as abap.char(8) )
cast( '19891109' as abap.int4 )
cast( '19891109' as abap.dec(10,2) )
cast( '19891109' as abap.dats )
```

- `abap.` prefix 필수
- Data Element로 CAST 가능 → **타입 + 의미 유지**

---

### 8. CASE Expressions in CDS

**Simple CASE**

```abap
case title
  when 'Mr.'  then 'Mister'
  when 'Mrs.' then 'Misses'
  else title
end as title_long
```

**Searched CASE**

```abap
case
  when seats_free = 0 then 'FULL'
  else 'AVAILABLE'
end as status
```

- ABAP SQL과 **문법·의미 동일**

---

### 9. Arithmetic Expressions in CDS

| 연산자 | 설명 |
| --- | --- |
| + | 덧셈 |
| - | 뺄셈 |
| * | 곱셈 |
| / | 나눗셈 (FLTP만 가능) |

**중요 제한**

- `/` 연산은 **Floating Point만 허용**
- 필요 시 `CAST( … as abap.fltp )` 사용

---

### 10. 핵심 한 줄 요약

> Code Pushdown은 계산 로직을 CDS View로 이동해
성능·재사용성·가독성을 동시에 확보하는 설계 방식이다.
> 

# Calling Built-in Functions in CDS Views  이론

### 1. Built-in Functions 개요

- ABAP SQL에서 쓰던 **내장 함수 대부분을 CDS View에서도 사용 가능**
- 단, **함수 종류별로 지원 범위가 다름**
    - Numeric: 거의 전부 지원
    - String: 대부분 지원 (일부 제외)
    - Date/Time: 제약 많음

---

### 2. Numeric Functions

✔ **전부 CDS에서 지원**

| 함수 | 설명 |
| --- | --- |
| DIV( ) | 정수 나눗셈 |
| MOD( ) | 나머지 |
| DIVISION( ) | 소수점 처리 나눗셈 |
| ROUND( ) | 반올림 |

**예시**

```abap
div( seats_occupied * 100, seats_max )       as percentage_int,
division( seats_occupied * 100, seats_max, 2 ) as percentage_dec
```

→ ABAP SQL과 **완전히 동일한 사용법**

---

### 3. String Processing Functions

✔ **대부분 지원**

❌ **연산자 `&&`는 CDS에서 사용 불가**

| 함수 | 지원 여부 |
| --- | --- |
| CONCAT( ) | O |
| CONCAT_WITH_SPACE( ) | O |
| UPPER( ) / LOWER( ) | O |
| SUBSTRING( ) | O |
| `&&` 연산자 | X |

**CDS에서는 반드시 함수 사용**

```abap
concat(
  street,
  concat_with_space( ',', postal_code, 1 )
) as address
```

---

### 4. Date & Time Functions (중요)

❌ **Generic 함수 사용 불가**

✔ **Type-specific 함수만 사용 가능**

| ABAP SQL 함수 | CDS에서 대체 |
| --- | --- |
| IS_VALID( ) | DATS_IS_VALID( ) |
| ADD_DAYS( ) | DATS_ADD_DAYS( ) |
| DAYS_BETWEEN( ) | DATS_DAYS_BETWEEN( ) |
| EXTRACT_MONTH( ) | 지원 안 됨 |
| WEEKDAY( ) | 지원 안 됨 |

📌 날짜 필드 타입이 `DATS`면 **DATS_*** 계열 사용

---

### 5. Date 정보 추출 우회 방법

- CDS에서는 `EXTRACT_*` 계열 함수 미지원
- **문자열 함수로 우회**

```abap
substring( flight_date, 5, 2 ) as month
```

→ 날짜를 문자열로 보고 잘라서 사용

---

### 6. Conversion Functions (예: 통화 변환)

### ABAP SQL

- 파라미터 이름 사용
- `=` 로 할당
- 타입 체크 느슨함

### ABAP CDS

- 파라미터 이름 사용 ❌
- `=>` 연산자 사용
- **타입 엄격 (CAST 필수)**

```abap
currency_conversion(
  amount              => total_price,
  source_currency     => currency_code,
  target_currency     => cast( 'EUR' as abap.cuky ),
  exchange_rate_date  => begin_date
) as total_price_eur
```

---

### 7. Timestamp Conversion (중요 차이)

- ABAP SQL: **Optional parameter 존재**
- ABAP CDS: **모든 파라미터 필수**

```abap
tstmp_to_dats(
  cast( lastchangedat as abap.dec(15,0) ),
  cast( 'EST' as abap.char(6) ),
  cast( '001' as abap.clnt ),
  'FAIL'
) as date_est
```

📌 CDS에서는 **unnamed parameters + 전부 mandatory**

---

### 8. Session Variables

- CDS에서 **ABAP 시스템 필드 접근용**
- `$session` prefix 사용

| Session Variable | 대응 ABAP 필드 |
| --- | --- |
| $session.client | sy-mandt |
| $session.system_date | sy-datum |
| $session.system_language | sy-langu |
| $session.user | sy-uname |
| $session.user_timezone | sy-zonlo |

**사용 예**

```abap
where FlightDate > $session.system_date
```

---

### 9. Session Variable 사용 시 주의

- 가능하면 **Input Parameter + Annotation 방식 권장**
- `$session.*`은 간편하지만 남용 비추천

### 10. 핵심 요약

- Numeric 함수: **전부 OK**
- String 함수: **연산자 ❌, 함수만 사용**
- Date/Time 함수: **Type-specific만 가능**
- Conversion 함수: **CDS는 타입 엄격**
- Session Variable: **ABAP 시스템 필드 대체 수단**

```csharp
@AbapCatalog.viewEnhancementCategory: [#NONE]
@AccessControl.authorizationCheck: #NOT_REQUIRED
@EndUserText.label: 'Employee (Consumption)'
@Metadata.ignorePropagatedAnnotations: true
@ObjectModel.usageType:{
    serviceQuality: #D,
    sizeCategory:   #M,
    dataClass:      #MASTER
}
define view entity zg01_c_employeequery
  as select from /LRN/R_Employee_Rel
{
  key EmployeeId,
      FirstName,
      LastName,
      DepartmentId,

      _Department.Description         as DepartmentDescription,
      _Department._assistant.LastName as AssistantName,

        @EndUserText.label: 'Employee Role'
        case EmployeeId
          when _Department.DepartmentHead    then 'H'
          when _Department.DepartmentAssistant then 'A'
          else ' '
        end as EmployeeRole,

        @EndUserText.label: 'Monthly Salary'
        @Semantics.amount.currencyCode: 'CurrencyCode'
        cast( AnnualSalary as abap.fltp ) / 12.0 as MonthlySalary,
        CurrencyCode,

      /* Associations */
      _Department
}

```

# 1

```csharp
        @EndUserText.label: 'Employee Role'
        case EmployeeId
          when _Department.DepartmentHead    then 'H'
          when _Department.DepartmentAssistant then 'A'
          else ' '
        end                                      as EmployeeRole,
```

**zg01_r_department 뷰에 *실제 컬럼명 : DepartmentHead /DepartmentAssistant 확인 필수* !**

→ 현재 직원(EmployeeId)이, 그 직원이 속한 부서의 부서장/비서인지 체크해서 ‘H’ 또는 ‘A’를 찍어주는 컬럼=이 계산 결과 컬럼의 이름을 **EmployeeRole**로 만든다.

# 2

```csharp

@EndUserText.label: 'Monthly Salary'
@Semantics.amount.currencyCode: 'CurrencyCode' "이 값은 금액(amount) 이고, 통화코드는 CurrencyCode 컬럼을 써라
cast( AnnualSalary as abap.fltp ) / 12.0 as MonthlySalary,
CurrencyCode,
```

→ 연봉(AnnualSalary)을 12로 나눠서 월급을 계산하고`MonthlySalary` 라는 **새 컬럼에 넣기**, 이 값이 어떤 통화인지(CurrencyCode:기존 컬럼을 그대로 노출하는 것)도 함께 붙여준다.
