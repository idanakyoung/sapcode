```csharp
 @EndUserText.label: 'Annual Salary'
        @Semantics.amount.currencyCode: 'CurrencyCodeUSD'
        currency_conversion( amount=> AnnualSalary,
                             source_currency => CurrencyCode,
                             target_currency => $projection.CurrencyCodeUSD,
                             exchange_rate_date => $session.system_date
                           )                as AnnualSalaryConverted,
```

 연봉을 USD로 환산(AnnualSalaryConverted)

`currency_conversion` 함수로 금액을 환산한다.

- `amount` : 원본 연봉(AnnualSalary)
- `source_currency` : 원본 통화코드(CurrencyCode)
- `target_currency` : 목표 통화(여기서는 USD). `$projection.CurrencyCodeUSD`를 사용한다는 점이 핵심.
- `exchange_rate_date` : 환율 적용일(세션의 시스템 날짜)

→ 주석 처리 후 그 자리 아래 코드 입력

```csharp
cast( $projection.AnnualSalaryConverted as abap.fltp )
        / 12.0                              as MonthlySalaryConverted,
```

월급(USD 기준) 계산(MonthlySalaryConverted)

월급을 USD로 환산된 연봉 / 12 로 만든다.

- `$projection.AnnualSalaryConverted`를 쓰는 이유 : 같은 SELECT 리스트 안에서 이미 계산한 컬럼을 다시 참조할 때 `$projection`으로 접근한다.
- `cast(... as abap.fltp)`를 하는 이유 : 정수/고정소수 타입으로 나눗셈하면 소수 처리 문제가 생길 수 있으니 부동소수로 바꿔서 계산 정확도를 확보하려는 의도

# Defining CDS Views with Input Parameters 이론

## 1. Lesson Objectives

이 레슨을 통해 다음을 할 수 있어야 한다.

- Input Parameter를 가지는 CDS View를 정의할 수 있다.
- CDS View Parameter에 값을 전달할 수 있다.
- CDS View Parameter를 ABAP 시스템 필드와 연결할 수 있다.

## 2. Input Parameters가 필요한 이유

이전 레슨에서 CDS View가 데이터베이스 레벨에서 얼마나 강력한 계산을 수행할 수 있는지 확인했다.

하지만 여기서 한 가지 한계가 있다.

- Consumer는 WHERE 절로 결과를 필터링할 수는 있지만
- CDS View 내부에서 수행되는 계산 로직 자체에는 영향을 주기 어렵다

이 한계를 해결하기 위해 **CDS View Input Parameters**가 필요하다.

### Input Parameters의 주요 목적

CDS View에 Input Parameters를 사용하면,

Consumer가 “조회 조건”이 아니라 **계산 방식 자체에 영향을 주는 값**을 전달할 수 있다.

대표적인 사용 사례는 다음과 같다.

- 통화 변환 시, 목표 통화를 Consumer가 지정
- 가격 계산 시, 세금 포함 여부를 Consumer가 지정
- 문자열 결합 시, 구분자(separator)를 Consumer가 지정
- 다국어 텍스트 조회 시, 언어 키를 Consumer가 지정

### Aggregation 이전 필터링

Aggregation이 있는 CDS View에서는

Aggregation 전에 어떤 데이터만 집계 대상에 포함할지 결정해야 하는 경우가 많다.

예시:

- 총 매출 계산 시, “특정 날짜 이후의 데이터만” 포함하고 싶을 때
- 이 날짜를 Input Parameter로 받아서 Aggregation 이전에 필터링

### Mandatory Selection 강제

SQL 자체는 “필수 선택 조건”이라는 개념을 모른다.

Consumer가 WHERE 조건을 줄 수도 있고, 안 줄 수도 있다.

하지만 Input Parameter를 사용하면:

- 특정 값(예: 언어, 기준 날짜, 사용자)을 **반드시 입력하도록 강제**할 수 있다.

## 3. Input Parameter 선언 방법

### 기본 문법

Input Parameter는 CDS View Entity 이름 뒤에

`WITH PARAMETERS` 구문으로 선언한다.

```abap
define view entity Z_Example
  with parameters
    p_language     : spras,
    p_targetCurr   : waers
as select from ...
```

### 선언 규칙

- 파라미터는 콤마(,)로 구분
- 각 파라미터는 반드시 타입을 가져야 함
- 이름과 타입은 콜론(:)으로 구분

### 허용되는 타입

- ABAP Dictionary의 기본 타입
    
    예: `abap.char(1)`, `abap.int4`
    
- Data Element
- Domain 기반 타입

허용되지 않는 타입:

- Structure 타입
- Table 타입
- Reference 타입

### 파라미터 어노테이션

Input Parameter에도 어노테이션을 붙일 수 있다.

```abap
@EndUserText.label: 'Target Currency'
p_targetCurr : waers
```

이는 Data Preview, UI, 문서화 시 사용된다.

## 4. CDS View 내부에서 Parameter 접근 방법

CDS View 정의 내부에서 Input Parameter에 접근할 때는

반드시 `$parameters.` prefix를 사용해야 한다.

```
$parameters.parameter_name
```

## 5. Parameter를 사용할 수 있는 위치

Input Parameter는 다음 위치에서 사용할 수 있다.

### 1) Element List (Select List)

파라미터를 하나의 컬럼처럼 직접 노출할 수 있다.

```abap
$parameters.TargetCurrency as CurrencyCode
```

주 사용 목적:

- 다른 컬럼의 `@Semantics.amount.currencyCode`에서 참조

### 2) CASE 구문

Consumer의 선택에 따라 계산 결과를 바꿀 수 있다.

```abap
case $parameters.FreeOrOccupied
  when 'X' then MaximumSeats - OccupiedSeats
  else OccupiedSeats
end as Seats
```

Consumer가 어떤 값을 넣느냐에 따라 결과 컬럼 의미가 달라진다.

### 3) Expressions / Functions

Built-in Function의 입력값으로 사용 가능하다.

```abap
currency_conversion(
  amount          => Price,
  source_currency => CurrencyCode,
  target_currency => $parameters.TargetCurrency,
  exchange_rate_date => FlightDate
) as PriceConverted
```

대표적인 사용 사례:

- 통화 변환
- 날짜 계산
- 문자열 처리

### 4) WHERE 절 조건

Input Parameter를 WHERE 조건의 우변에 사용 가능하다.

```abap
where FlightDate > $parameters.FirstFlightDate
```

이 경우, 해당 파라미터는 사실상 **Mandatory Selection 조건** 역할을 한다.

### 5) Path Expression의 Filter 조건

Association 경로의 필터 조건에서도 사용 가능하다.

```abap
_Currency._Text[ Language = $parameters.Language ].CurrencyName
```

다국어 텍스트 조회에서 매우 자주 사용된다.

## 6. Data Preview에서 Input Parameters

Input Parameter를 정의한 CDS View를

Eclipse의 Data Preview로 열면, 먼저 파라미터 입력 화면이 나타난다.

### 특징

- 파라미터 이름 옆에  표시가 있으면 Mandatory
- 입력값은 파라미터 타입과 호환되어야 함
- 타입 체크 외의 추가 검증은 수행하지 않음

입력값을 모두 넣어야 Data Preview가 실행된다.

## 7. View-on-View에서 Input Parameters 사용

Input Parameter를 가진 CDS View는

다른 CDS View의 데이터 소스로도 사용 가능하다.

### 문법

```abap
as select from Z_Base_View(
  param1 : 'X',
  param2 : $parameters.param2
)
```

### 특징

- 파라미터는 괄호 `()` 안에 전달
- 파라미터 할당 연산자는 콜론(:)
- 리터럴 값 전달 가능
- 상위 View의 파라미터를 그대로 전달(Forwarding) 가능

Forwarding을 사용하면:

- 최상위 Consumer만 실제 값을 제공
- 중간 CDS View들은 파라미터를 전달만 함
- CDS View 스택 전체로 값이 전달됨

Eclipse에서는 `Ctrl + Space`로

필요한 파라미터 시그니처를 자동 완성할 수 있다.

## 8. ABAP SQL에서 CDS View Parameter 전달

ABAP SQL에서 Input Parameter를 가진 CDS View를 사용할 수 있다.

### CDS와 ABAP SQL의 차이점

- CDS: 파라미터 할당에 콜론(:) 사용
- ABAP SQL: 파라미터 할당에 등호(=) 사용

### 1) Literal 값 사용

```abap
SELECT FROM Z_View(
  p_date = '20230101',
  p_lang = 'D'
)
FIELDS ...
```

### 2) Host Variable 사용

```abap
SELECT FROM Z_View(
  p_date = @lv_date,
  p_lang = @lv_lang
)
FIELDS ...
```

- Host Variable은 `@`로 escape
- 타입은 파라미터 타입과 호환되어야 함
- Elementary 타입만 허용

### 3) Host Expression 사용

```abap
SELECT FROM Z_View(
  p_date = @( cl_abap_context_info=>get_system_date( ) ),
  p_lang = @( cl_abap_context_info=>get_user_language_iso_format( ) )
)
FIELDS ...
```

- `@(` `)`로 감싼 ABAP Expression 사용 가능
- Expression 결과는 반드시 Elementary 타입이어야 함

## 9. CDS Parameter와 ABAP System Field 연결

CDS View Parameter는

ABAP 시스템 필드 값을 자동으로 받을 수 있다.

### 설정 방법

1. 시스템 필드와 타입이 맞는 Input Parameter 정의
2. 해당 파라미터에 `@Environment.systemField` 어노테이션 지정

```abap
with parameters
  @Environment.systemField: #SYSTEM_LANGUAGE
  Language : spras,

  @Environment.systemField: #SYSTEM_DATE
  BeginDate : dats
```

### 동작 방식

- ABAP SQL에서 값을 명시적으로 전달하지 않으면
- ABAP Runtime이 해당 시스템 필드 값을 자동으로 주입
- CDS/DB 레벨에서는 여전히 Mandatory Parameter

즉,

- ABAP에서 사용할 때만 “기본값처럼 보이는 효과”
- CDS 자체에서는 Optional Parameter가 아님

## 10. 지원되는 systemField 값

`@Environment.systemField` 어노테이션은 다음 값을 지원한다.

| Annotation Value | ABAP System Field |
| --- | --- |
| #CLIENT | sy-mandt |
| #SYSTEM_DATE | sy-datum |
| #SYSTEM_TIME | sy-uzeit |
| #SYSTEM_LANGUAGE | sy-langu |
| #USER | sy-uname |
| #USER_DATE | sy-datlo |
| #USER_TIMEZONE | sy-zonlo |

## 11. 주의사항

- ABAP Cloud 환경에서는
    
    일부 시스템 필드의 직접 접근이 권장되지 않는다.
    
- 이 경우 `CL_ABAP_CONTEXT_INFO`의 Getter 메서드 사용이 권장된다.
- systemField와 연결된 파라미터라도
    
    CDS 및 DB 레벨에서는 여전히 Mandatory로 취급된다.
    

## 12. Lesson Summary

이 레슨에서 배운 내용은 다음과 같다.

- Input Parameter를 가지는 CDS View를 정의할 수 있다.
- 다양한 위치(Element List, CASE, Function, WHERE 등)에서 Parameter를 사용할 수 있다.
- View-on-View, ABAP SQL에서 Parameter를 전달할 수 있다.
- CDS View Parameter를 ABAP 시스템 필드와 연결할 수 있다.
