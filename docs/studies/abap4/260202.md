# 📌 ABAP `REPLACE` 함수 정리

## 1️⃣ 기본 문법

```abap
result = REPLACE(
  VAL  = source
  SUB  = substring
  [CASE = abap_bool]
  WITH = new_string
).

```

- `VAL` : 원본 문자열
- `SUB` : 바꿀 대상 문자열
- `WITH` : 대체할 문자열
- `CASE` : 대소문자 구분 여부 (선택)

---

## 2️⃣ CASE 옵션 정리

| CASE 값 | 의미 |
| --- | --- |
| `ABAP_TRUE` | 대소문자 **구분함** |
| `ABAP_FALSE` | 대소문자 **구분 안 함** (기본값) |

👉 `CASE`를 생략하면 자동으로 **ABAP_FALSE**

## 1️⃣ `| |` : String Template

```abap
|{ ls_line-bookid }|

```

- `| |` : 문자열 템플릿
- `{ }` : **ABAP Expression**
- 안에 변수, 계산, 포맷 옵션 다 가능

👉 `WRITE`, `CONCATENATE` 대체

---

## 2️⃣ `&&` : 문자열 연결 연산자

```abap
| ... | && | ... |

```

- 문자열 + 문자열 연결
- `CONCATENATE`보다 가독성 좋음
- Modern ABAP 스타일

---

## 3️⃣ `{ iv_separator }` : 구분자 처리

```abap
{ iv_separator }

```

- 보통 `;` 또는 `|`
- CSV / 다운로드 파일용
- **하드코딩 ❌ / 파라미터화 ⭕**

👉 실무 포인트 👍

---

## 4️⃣ 클래스 코드 (`lv_class`)

```abap
{ lv_class }

```

- 앞에서 `Y → ECO`, `C → BUS` 로 변환된 값
- **DB 코드 → 사용자 친화 코드**

---

## 5️⃣ 💰 금액 포맷 (중요 ⭐)

```abap
{ ls_line-loccuram
  CURRENCY = ls_line-loccurkey
  NUMBER   = USER }

```

### 의미

- `CURRENCY =` : 통화 단위 적용
- `NUMBER = USER` : 사용자 숫자 포맷 (소수점, 콤마)

👉 예전 `WRITE ... TO lv_price` 방식 대체

📌 **시험 포인트**

> String Template에서도 통화/숫자 포맷 가능
> 

---

## 6️⃣ 날짜 포맷

```abap
{ ls_line-order_date DATE = USER }

```

- 사용자 로케일 기준 날짜 포맷
- `YYYYMMDD` → `DD.MM.YYYY` 등

👉 `WRITE TO lv_date` 필요 없음

📌 이유

- 포맷 옵션(`CURRENCY`, `DATE`)은

### Excercise 2 (**FIELD-SYMBOL + ASSIGNING**)

```jsx
REPORT zd750aw_expr2_g29.

CLASS lcl_flights DEFINITION.
  PUBLIC SECTION.
    TYPES tt_flights TYPE STANDARD TABLE OF sflight WITH NON-UNIQUE KEY carrid connid fldate.
    CLASS-METHODS get_flights IMPORTING iv_carrid      TYPE sflight-carrid
                                        iv_connid      TYPE sflight-connid
                              RETURNING VALUE(rt_list) TYPE d75aw_t_flights.
  PRIVATE SECTION.
    CLASS-METHODS read_flights IMPORTING iv_carrid         TYPE sflight-carrid
                                         iv_connid         TYPE sflight-connid
                               RETURNING VALUE(rt_flights) TYPE tt_flights.

ENDCLASS.

CLASS lcl_flights IMPLEMENTATION.
  METHOD get_flights.
    DATA:       ls_result  TYPE d75aw_s_flight.
*          lt_flights TYPE tt_flights,
*
*          lx_error   TYPE REF TO cx_root.
*    FIELD-SYMBOLS <line> TYPE sflight.

    DATA(lt_flights) = read_flights( iv_carrid = iv_carrid iv_connid = iv_connid ).
    LOOP AT lt_flights ASSIGNING FIELD-SYMBOL(<line>).
      MOVE-CORRESPONDING <line> TO ls_result.
      TRY.
          ls_result-percentage = ls_result-seatsocc / ls_result-seatsmax * 100.
          APPEND ls_result TO rt_list.
        CATCH cx_sy_zerodivide INTO DATA(lx_error).
*       Some suitable error processing here
      ENDTRY.
    ENDLOOP.
  ENDMETHOD.

  METHOD read_flights.
    SELECT * FROM sflight INTO TABLE rt_flights WHERE carrid = iv_carrid AND connid = iv_connid.
  ENDMETHOD.

ENDCLASS.

*DATA: gt_flight_list TYPE d75aw_t_flights,
*      go_alv         TYPE REF TO cl_salv_table.

PARAMETERS: pa_car TYPE sflight-carrid DEFAULT 'LH',
            pa_con TYPE sflight-connid DEFAULT '0400'.

START-OF-SELECTION.
  DATA(gt_flight_list) = lcl_flights=>get_flights(
                       iv_carrid =  pa_car
                       iv_connid = pa_con ).

  cl_salv_table=>factory(
    IMPORTING
      r_salv_table   =  DATA(go_alv)   " Basis Class Simple ALV Tables
    CHANGING
      t_table        = gt_flight_list ).

  go_alv->display( ).
```

## 6️⃣ 왜 STRING TABLE을 쓰는가?

### 🔹 STRING 한 줄의 단점

- 조건 추가/삭제가 번거로움
- AND / OR 처리 실수 잦음

### 🔹 STRING TABLE의 장점

- 조건을 **조각 단위로 관리**
- IF 조건으로 APPEND 여부 제어 가능
- 실무에서 훨씬 안전함

```abap
IF pa_car IS NOT INITIAL.
  APPEND |carrid = '{ pa_car }'| TO gt_where.
ENDIF.

IF pa_con IS NOT INITIAL.
  APPEND |and connid = '{ pa_con }'| TO gt_where.
ENDIF.

```

1. SAP 표준 도메인/데이터요소 `S_CONN_ID` 타입 : 보통 `NUMC 4` 기반이라 길이 4의 숫자 문자열 형태를 기대함. `NUMC`는 내부적으로 문자처럼 저장되지만 숫자만 들어오길 기대하는 필드

4. `DATA(GX_ROOT)`는 **inline 선언** 문법으로, *DATA: GX_ROOT TYPE REF TO CX_ROOT. 이렇게 주석 처리되어 있어도 정상 작동
5. `EXACT #( )`는 ABAP의 **엄격 타입 변환 표현식**
6. CATCH CX_SY_CONVERSION_ERROR INTO DATA(GX_EXCP).
    1. **형 변환(conversion) 오류만** 잡는 CATCH
    2. INTO DATA(gx_excp)  : inline 선언 (**지역 변수** 생성)

    그래서 `GET_TEXT( ) , GET_LONGTEXT( ) , GET_SOURCE_POSITION( )`같은 메서드를 사용할 수 있음

8. CATCH CX_ROOT INTO GX_ROOT.
    1. 모든 클래스 기반 예외를 잡음
    2. `cx_sy_conversion_error`도 여기의 하위 클래스
    3. 전제 조건 : DATA gx_root TYPE REF TO cx_root. 위쪽에 반드시 선언 / 인라인 X
9. CATCH CX_ROOT INTO DATA(GX_ROOT).  → 실사용 코드
    1. **모든 예외를 잡되** inline 선언 방식으로 `gx_root` 생성
10. DATA(lv_msg) = gx_root->get_text( ). 
    1. `gx_root`는 예외 객체 참조
    2. get_text( ) : SAP 표준 예외 메시지(짧은 설명)를 문자열로 반환
    3. `lv_msg`는 inline 선언된 STRING 계열 변수

1. 방식 1 : 단일 문자열(`gv_where`)로 where 조건 만드는 방식
    1. 이 방식 특징 
    - where 조건을 **한 줄 문자열로 직접 완성**
    - 조건이 단순할 때는 코드가 짧고 직관적
    - 조건이 많아지면 문자열 관리가 어려워짐
    - 조건 on/off 분기 처리하기 불편함
    
    ```csharp
    REPORT zabap_61_g01.
    
    DATA: gv_where    TYPE string,          " where 조건을 한 줄 문자열로 저장
          gt_flights  TYPE TABLE OF sflight." 조회 결과 저장용 internal table
    
    PARAMETERS: pa_car TYPE sbook-carrid,    " 항공사 코드 입력
                pa_con TYPE sbook-connid.    " 항공편 번호 입력
    
    *---------------------------------------------------------------------*
    * [방식 1] 단일 string으로 where 조건 생성
    *---------------------------------------------------------------------*
    
    " string template을 이용해 where 조건을 한 번에 완성
    " 사용자가 입력한 값이 { } 안에 들어감
    " 예: carrid = 'LH' and connid = '0400'
    gv_where = |carrid = '{ pa_car }' and connid = '{ pa_con }'|.
    
    SELECT *
      INTO TABLE gt_flights
      FROM sflight
      WHERE (gv_where).   " 단일 문자열을 동적 where 조건으로 사용
    
    cl_demo_output=>display( gt_flights ).
    
    ```
    
2. 방식 2 : 문자열 테이블(`gt_where`)로 where 조건 만드는 방식
    1. 이 방식 특징
    - where 조건을 **여러 줄로 나누어 관리**
    - 조건 추가/삭제가 쉬움
    - if 문으로 조건 분기하기 좋음
    - 실무에서 더 선호되는 방식
    
    ```csharp
    REPORT zabap_61_g01.
    
    DATA: gv_where    TYPE string,           " where 조건 한 줄용 변수

                pa_con TYPE sbook-connid.     " 항공편 번호 입력
    
    *---------------------------------------------------------------------*
    * [방식 2] 문자열 internal table로 where 조건 생성
    *---------------------------------------------------------------------*
    
    " 첫 번째 조건 생성
    " 예: carrid = 'LH'
    gv_where = |carrid = '{ pa_car }'|.
    APPEND gv_where TO gt_where.
    
    " 두 번째 조건을 별도의 줄로 추가
    " 앞에 and를 붙여서 sql 조건 연결
    " 예: and connid = '0400'
    APPEND | and connid = '{ pa_con }'| TO gt_where.
    
    SELECT *
      INTO TABLE gt_flights
      FROM sflight

    cl_demo_output=>display( gt_flights ).
    ```
