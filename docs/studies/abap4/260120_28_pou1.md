```csharp
*& <--  p2        text

FORM SET_DEFAULT_FLIGHT_DATE .
  DATA LV_HIGH TYPE SY-DATUM.
  LV_HIGH = SY-DATUM.

  CALL FUNCTION 'RP_CALC_DATE_IN_INTERVAL'
    EXPORTING
      DATE      = LV_HIGH
      MONTHS    = 6
      SIGNUM    = '+'
      DAYS      = 0
      YEARS     = 0
    IMPORTING
      CALC_DATE = LV_HIGH.

  CLEAR SO_FDT[].
  SO_FDT-SIGN   = 'I'.
  SO_FDT-OPTION = 'BT'.
  SO_FDT-LOW    = SY-DATUM.
  SO_FDT-HIGH   = LV_HIGH.
  APPEND SO_FDT.

ENDFORM.

*& Form LOCK_FLDATE_LOW
*&---------------------------------------------------------------------*
*& text
*&---------------------------------------------------------------------*
*& -->  p1        text
*& <--  p2        text

FORM LOCK_FLDATE_LOW .
  LOOP AT SCREEN.
    IF SCREEN-NAME = 'SO_FDT-LOW'.
      SCREEN-INPUT = 0.  "Output only.
      MODIFY SCREEN.
    ENDIF.
  ENDLOOP.
ENDFORM.
*&---------------------------------------------------------------------*
*& Form SET_LISTBOX_VALUES
*&---------------------------------------------------------------------*
*& text
*&---------------------------------------------------------------------*
*& -->  p1        text
*& <--  p2        text

FORM SET_LISTBOX_VALUES .

  GS_VRM-KEY  = 'P'.
  GS_VRM-TEXT = 'Private'.
  APPEND GS_VRM TO GT_VRM.

  GS_VRM-KEY  = 'B'.
  GS_VRM-TEXT = 'Business'.
  APPEND GS_VRM TO GT_VRM.

  CALL FUNCTION 'VRM_SET_VALUES'
    EXPORTING
      ID     = 'P_BPCUST'
      VALUES = GT_VRM.
ENDFORM.
```

### Level 2 고객별 예약 현황 조회 (Summary ALV)

### 1. Summary ALV 출력 구조

고객 단위 집계를 위해 별도의 Summary 구조를 정의한다.

- 고객 기본 정보 (Customer ID, Name, 주소, 연락처 등)
- 예약 건수 (BOOKING_CNT)
- 예약 건수 기준 고객 등급 (Silver / Gold / Platinum)
- 상태 표시용 Light 필드
- ALV 색상 처리를 위한 CTAB 필드

```abap
TYPES: BEGIN OF TS_SUM,
         LIGHT       TYPE C LENGTH 1,
         GRADE       TYPE CHAR10,
         CTAB        TYPE LVC_T_SCOL,
         CUSTID      TYPE SCUSTOM-ID,
         NAME        TYPE SCUSTOM-NAME,
         CUSTTYPE    TYPE SCUSTOM-CUSTTYPE,
         BP_TEXT     TYPE CHAR20,
         BOOKING_CNT TYPE I,
         POSTCODE    TYPE SCUSTOM-POSTCODE,
         STREET      TYPE SCUSTOM-STREET,
         CITY        TYPE SCUSTOM-CITY,
         DISCOUNT    TYPE SCUSTOM-DISCOUNT,
         TELEPHONE   TYPE SCUSTOM-TELEPHONE,
         EMAIL       TYPE SCUSTOM-EMAIL,
       END OF TS_SUM.
```

### 2. 개발 목적

Selection Screen에서 입력한 조건을 기준으로 고객별 예약 건수를 집계하여 Summary ALV로 출력한다.

예약 건수에 따라 고객 등급과 상태를 표시하고, Application Toolbar의 Booking List 버튼을 통해 다음 화면으로 이동할 수 있도록 한다.

### 4. 데이터 조회 로직

```csharp
SELECT CUSTOMID COUNT( * ) AS CNT
    INTO TABLE LT_CNT
    FROM SBOOK
    WHERE CUSTOMID IN SO_CUST
      AND FLDATE   IN SO_FDT
      AND CLASS IN RT_CLASS
    GROUP BY CUSTOMID.

  SELECT *
    FROM SCUSTOM
    INTO TABLE LT_SCUSTOM
    FOR ALL ENTRIES IN LT_CNT
    WHERE ID = LT_CNT-CUSTOMID.

  CASE P_BPCUST.
    WHEN 'P'.
      DELETE LT_SCUSTOM WHERE CUSTTYPE <> 'P'.
    WHEN 'B'.
      DELETE LT_SCUSTOM WHERE CUSTTYPE <> 'B'.
    WHEN OTHERS.
  ENDCASE.
```

### 5. 파생 데이터 생성

```csharp
* 파생필드 채우기.

  LOOP AT LT_SCUSTOM INTO LS_SCUSTOM.

    CLEAR GS_SUM.

    "BASIC
    GS_SUM-CUSTID    = LS_SCUSTOM-ID.

    CASE GS_SUM-CUSTTYPE.
      WHEN 'P'. GS_SUM-BP_TEXT = 'Private'.
      WHEN 'B'. GS_SUM-BP_TEXT = 'Business'.
      WHEN OTHERS. GS_SUM-BP_TEXT = ''.

    "BOOKING_CNT

      GS_SUM-BOOKING_CNT = 0.
```

- 풀이
    
    ## 행동 순서도(전체 플로우)
    
    1. **Selection Screen(레벨1)**
        
        → 조건 입력(고객, 항공일자(6개월후 기본), Class, B/P Dropdown)
        
        → 실행(F8)
        
    2. **고객별 예약 현황 ALV(레벨2)**
        
        → 고객별 예약건수 집계(취소 제외)
        
        → Status 색(실버/골드/플래티넘) + 총합(Booking Counter Total)
        
        → 버튼 **Booking List** 클릭
        
    - 선택없음: 메시지
    - 선택있음: 고객 예약 리스트로 이동
    1. **고객 예약 정보 ALV(레벨3)**
        
        → 선택 고객의 SBOOK 예약 리스트(Flight date 조건, 취소 제외)
        
        → 기내식 신청 상태를 **아이콘(회색/초록/빨강)** 으로 표시
        
        → **Request/Change** 버튼: 메뉴 신청/변경 화면 이동
        
        → **Cancel(셀 버튼)**: 신청 상태 R일 때만 표시 + Confirm 팝업 후 C로 변경
        
        → **Hotspot**: Connid/FLDATE/Menu 클릭 시 팝업(레벨5~6)
        
    2. **기내식 신청/변경 화면(레벨4)**
        
        → 상단 Booking Info(항공사/편명/일자/예약번호/요청일자 기본=오늘)
        
        → 하단: 해당 항공사 **SMENU** 목록 ALV
        
        → 선택 후 **SAVE**
        
    - 기존 신청 없으면 생성
    - 기존 신청 있고 같은 메뉴면 메시지
    - 기존 신청 있고 다른 메뉴면 변경(REQDAT=오늘, REQSTS=R)
    1. **기내식 신청 내역 팝업(레벨5: Hotspot)**
        
        → Menu(또는 메뉴 관련 Hotspot) 클릭 시
