## Level 1 - 조회(Selection Screen → ALV 출력)

## 레슨 1) ALV에 버튼(신청/변경/취소) 만들고 이벤트 받기

### 1-1. PF-STATUS에 버튼 추가

지금 `STATUS_0100`에서 `SET PF-STATUS 'S100'.` 쓰고 있으니, GUI Status `S100`에

- `APPLY` (신청)
- `CHANGE` (변경)
- `ACANCEL` (취소)
    
    를 Function code로 만들어서 버튼으로 올려.
    

### 1-2. “선택된 Row 가져오기” 공통 루틴 만들기

변경/취소는 “Row 선택 안 하면 메시지”가 스펙에 있어. (p.4, p.5)

ALV에서 선택된 행을 가져오는 FORM 하나 만들어두면 편해.

```abap
FORM get_selected_row CHANGING cv_index TYPE i.
  DATA: lt_rows TYPE lvc_t_row,
        ls_row  TYPE lvc_s_row.

  CLEAR cv_index.

  CALL METHOD go_grid->get_selected_rows
    IMPORTING
      et_index_rows = lt_rows.

  READ TABLE lt_rows INTO ls_row INDEX 1.
  IF sy-subrc = 0.
    cv_index = ls_row-index.
  ENDIF.
ENDFORM.

```

그리고 `USER_COMMAND_0100`에서 `CHANGE/ACANCEL` 눌렀을 때:

```abap
DATA lv_idx TYPE i.

PERFORM get_selected_row CHANGING lv_idx.
IF lv_idx IS INITIAL.
  MESSAGE 'Data(Row)를 선택하여 주세요!' TYPE 'I'.
  EXIT.
ENDIF.

```

## 레슨 2) “신청” 팝업(모달) 화면 만들기 (Screen 200 추천)

요청서 p.3의 팝업 요구사항

- 신청자 사번 Search Help
- 취소 클릭 시 팝업 종료
- 저장 클릭 시 DB Insert
- 신청일 = 오늘
- 상태 = 'A'
- 신청번호 = “현재일+현재시간”

### 2-1. Screen 200(팝업) 만들기

- 화면 속성: **Modal Dialog Box** 체크
- 입력 필드 예시(전역 변수/구조에 바인딩)
    - 사번 (EMPNO) : Search help는 보통 Data Element에 붙어 있으면 자동
    - 제증명 종류 (CERT_KIND) : 리스트박스/드롭다운
    - 신청수량 (REQ_QTY)
    - 수령방법 (RCV_METHOD) : 라디오버튼(직접/우편)처럼 해도 됨(예시 화면이 라디오)
    - 용도 (CERT_USE) : 드롭다운

### 2-2. 팝업 모드 구분 변수

신청/변경 화면을 같은 팝업으로 재활용하려면 모드가 필요해.

```abap
DATA gv_mode TYPE c LENGTH 1. " 'A' 신청, 'U' 변경
DATA gs_pop  TYPE ztcerti_g01. " 팝업 입력용 (또는 별도 TY)

```

신청 버튼 눌렀을 때:

```abap
WHEN 'APPLY'.
  CLEAR: gs_pop.
  gv_mode = 'A'.
  CALL SCREEN 200 STARTING AT 10 3 ENDING AT 80 20.

```

## 레슨 3) “신청 저장” 로직 (INSERT) + 신청번호 만들기

요구: 신청번호 = 현재일 + 현재시간 (p.3)

간단/안전하게는 문자열로 만들고, DB 필드 타입(REQ_NO)이 NUM 14라면 “YYYYMMDDHHMMSS”는 14자리라 딱 맞아.

```abap
FORM make_reqno CHANGING cv_reqno TYPE ztcerti_g01-req_no.
  DATA lv_ts TYPE timestampl.
  GET TIME STAMP FIELD lv_ts.

  " YYYYMMDDHHMMSS (14)
  cv_reqno = |{ lv_ts TIMESTAMP = ISO }|.
  " 위 포맷이 환경 따라 다르면 아래 방식 추천:
  " cv_reqno = |{ sy-datum }{ sy-uzeit }|.  " (주의: uzeit 6자리)
ENDFORM.
```

팝업에서 저장 버튼(예: `SAVE`) 누르면:

```abap
FORM save_apply.
  DATA ls_ins TYPE ztcerti_g01.

  ls_ins-mandt      = sy-mandt.
  ls_ins-req_date   = sy-datum.
  ls_ins-empno      = gs_pop-empno.
  ls_ins-cert_kind  = gs_pop-cert_kind.
  ls_ins-req_qty    = gs_pop-req_qty.
  ls_ins-rcv_method = gs_pop-rcv_method.
  ls_ins-cert_use   = gs_pop-cert_use.
  ls_ins-req_status = 'A'.

  PERFORM make_reqno CHANGING ls_ins-req_no.

  INSERT ztcerti_g01 FROM ls_ins.
  IF sy-subrc = 0.
    COMMIT WORK AND WAIT.
    MESSAGE '신청이 저장되었습니다.' TYPE 'S'.
  ELSE.
    ROLLBACK WORK.
    MESSAGE '저장 실패' TYPE 'E'.
  ENDIF.
ENDFORM.
```

저장 후에는

- `LEAVE TO SCREEN 0.`
- 그리고 메인에서 `PERFORM get_data.` + `go_grid->refresh_table_display( )`
    
    까지 하면 UX 완성.

## 레슨 4) “변경” 팝업 + 변경 불가 상태 체크

요구사항(p.4)

- Row 미선택 시 I 메시지
- Row 선택 후 변경 클릭 시 **사번은 Output**
- 상태가 C/F/P 면 변경 불가 + “신청을 변경 할 수 없습니다.” I 메시지
- 저장 시 DB Update

### 4-1. 상태 체크 로직 (공통 함수처럼)

```abap
FORM check_change_allowed USING iv_status TYPE ztcerti_g01-req_status
                          CHANGING cv_ok TYPE abap_bool.
  cv_ok = abap_true.
  IF iv_status = 'C' OR iv_status = 'F' OR iv_status = 'P'.
    cv_ok = abap_false.
  ENDIF.
ENDFORM.

```

### 4-2. 변경 버튼 처리 흐름

```abap
WHEN 'CHANGE'.
  DATA lv_idx TYPE i.
  DATA lv_ok  TYPE abap_bool.

  PERFORM get_selected_row CHANGING lv_idx.
  IF lv_idx IS INITIAL.
    MESSAGE 'Data(Row)를 선택하여 주세요!' TYPE 'I'.
    EXIT.
  ENDIF.

  READ TABLE gt_alv INTO gs_alv INDEX lv_idx.

  PERFORM check_change_allowed USING gs_alv-req_status CHANGING lv_ok.
  IF lv_ok = abap_false.
    MESSAGE '신청을 변경 할 수 없습니다.' TYPE 'I'.
    EXIT.
  ENDIF.

  gv_mode = 'U'.
  " 팝업에 표시할 값 세팅
  MOVE-CORRESPONDING gs_alv TO gs_pop.

  CALL SCREEN 200 STARTING AT 10 3 ENDING AT 80 20.

```

팝업 Screen 200 PBO에서 `gv_mode = 'U'`면 EMPNO 필드를 `input = 0` 처리(화면 Painter에서 속성 바꾸거나 LOOP AT SCREEN로 제어).

## 레슨 5) “취소” 기능 + 확인 팝업(POPUP_TO_CONFIRM)

요구사항(p.6)

- Row 미선택 시 I 메시지
- 상태가 C/F/P 면 취소 불가 + “신청을 취소 할 수 없습니다.”
- 선택 후 취소 클릭 시 `POPUP_TO_CONFIRM`
- YES면 상태를 ‘C’로 업데이트

```abap
WHEN 'ACANCEL'.
  DATA lv_idx TYPE i.
  DATA lv_ok  TYPE abap_bool.
  DATA lv_ans TYPE c.

  PERFORM get_selected_row CHANGING lv_idx.
  IF lv_idx IS INITIAL.
    MESSAGE 'Data(Row)를 선택하여 주세요!' TYPE 'I'.
    EXIT.
  ENDIF.

  READ TABLE gt_alv INTO gs_alv INDEX lv_idx.

  " 취소 가능 여부 체크 (C/F/P 불가)
  lv_ok = abap_true.
  IF gs_alv-req_status = 'C' OR gs_alv-req_status = 'F' OR gs_alv-req_status = 'P'.
    lv_ok = abap_false.
  ENDIF.
  IF lv_ok = abap_false.
    MESSAGE '신청을 취소 할 수 없습니다.' TYPE 'I'.
    EXIT.
  ENDIF.

  CALL FUNCTION 'POPUP_TO_CONFIRM'
    EXPORTING
      text_question = '제증명 신청을 취소 하시겠습니까?'
      text_button_1 = 'Yes'
      text_button_2 = 'No'
    IMPORTING
      answer        = lv_ans.

  IF lv_ans = '1'. " Yes
    UPDATE ztcerti_g01
       SET req_status = 'C'
     WHERE req_no = gs_alv-req_no.
    IF sy-subrc = 0.
      COMMIT WORK AND WAIT.
      MESSAGE '취소되었습니다.' TYPE 'S'.
      PERFORM get_data.
      go_grid->refresh_table_display( ).
    ENDIF.
  ENDIF.

```
