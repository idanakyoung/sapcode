## LEVEL 0. TOP INCLUDE

이 레벨의 목적은 모든 Screen/PBO/PAI/FORM에서 같이 쓰는 변수/테이블/상태를 준비

## 코드: TOP 전체

```abap
*&---------------------------------------------------------------------*
*& Include SAPMZABAP2_G01_POU_02TOP                 - Module Pool      SAPMZABAP2_G01_POU_02
*&---------------------------------------------------------------------*
PROGRAM SAPMZABAP2_G01_POU_02.

TABLES: SBOOK, ZST_BOOKING_G01, ZST_CONN_G01, ZST_FLINFO_G01, ZST_CUST_G01, ZTSURVEY_G01, ZTMENUREQ_F01.

DATA : OK_CODE TYPE SY-UCOMM.

CONTROLS: TABSTRIP TYPE TABSTRIP.

DATA: GV_ACTTAB TYPE SY-UCOMM VALUE 'TAB1'.

DATA: GV_CARRID TYPE SBOOK-CARRID,
      GV_CONNID TYPE SBOOK-CONNID,
      GV_FLDATE TYPE SBOOK-FLDATE,
      GV_BOOKID TYPE SBOOK-BOOKID.

DATA: GS_BOOKING TYPE ZST_BOOKING_G01,
      GS_CONN    TYPE ZST_CONN_G01,    "TAB2
      GS_FLINFO  TYPE ZST_FLINFO_G01,  "TAB3
      GS_CUST    TYPE ZST_CUST_G01,    "TAB4
      GS_SURVEY  TYPE ZTSURVEY_G01.

TYPES: BEGIN OF TY_MENU_POP,
         MENUNUMBER TYPE S_MENU,      "Menu
         APP_ID     TYPE S_STARTER,   "Appetizer No
         APP_TXT    TYPE CHAR40,      "Appetizer Name
         MAIN_ID    TYPE S_MACOURSE,  "Main No
         MAIN_TXT   TYPE CHAR40,      "Main Name
         DES_ID     TYPE S_DESSERT,   "Dessert No
         DES_TXT    TYPE CHAR40,      "Dessert Name
       END OF TY_MENU_POP.

DATA: GS_MENU_POP TYPE TY_MENU_POP.

DATA: GV_MENU_EXISTS   TYPE ABAP_BOOL, " 존재 여부 플래그
      GV_SURVEY_EXISTS TYPE ABAP_BOOL.

DATA: GV_SRV_MODE TYPE C LENGTH 1. "  Screen 200 모드: C(Create) / D(Display)
```

## 설명(역할)

- `TABLES SBOOK`: Screen 0100에서 입력받는 예약 키(항공사/편명/일자/예약번호)
- `TABLES ZST_*`: Screen 0100의 각 탭 표시용 구조
- `TABLES ZTSURVEY_G01`: **Screen 0200 설문 화면 바인딩 + DB 저장/조회에 사용하는 핵심 워크영역**
- `OK_CODE`: 버튼/기능코드가 들어오는 변수(=SY-UCOMM)
- `TABSTRIP / GV_ACTTAB`: 탭스트립 현재 탭 유지
- `GS_*`: 조회 결과를 임시로 담는 구조체(특히 `GS_BOOKING`은 FM 결과)
- `GV_MENU_EXISTS / GV_SURVEY_EXISTS`: 버튼 노출을 결정하는 플래그
- `GV_SRV_MODE`: Screen 0200을 “작성/조회”로 전환하는 모드 스위치

## LEVEL 1. Screen 0100의 PBO(STATUS_0100) - 화면 뜰 때 버튼/탭 상태 결정

이 레벨은 “화면을 그리기 직전”에 실행돼.

여기서 **탭 활성화 유지**, **버튼 숨김 목록 생성**, **평균값 준비** 같은 UI 상태를 정리한다.

## 코드: STATUS_0100 전체

```abap
MODULE STATUS_0100 OUTPUT.
  DATA LT_EXCL TYPE TABLE OF SY-UCOMM.

  SET TITLEBAR 'T100'.

  IF GV_ACTTAB IS INITIAL.
    GV_ACTTAB = 'TAB1'.
  ENDIF.
  TABSTRIP-ACTIVETAB = GV_ACTTAB.

  CLEAR: GV_MENU_EXISTS, GV_SURVEY_EXISTS.
  IF ZST_BOOKING_G01-CARRID IS NOT INITIAL
   AND ZST_BOOKING_G01-CONNID IS NOT INITIAL
   AND ZST_BOOKING_G01-FLDATE IS NOT INITIAL
   AND ZST_BOOKING_G01-BOOKID IS NOT INITIAL.

    PERFORM CHECK_MENU_EXISTS  CHANGING GV_MENU_EXISTS.
    PERFORM CHECK_SURVEY_EXISTS CHANGING GV_SURVEY_EXISTS.
    PERFORM SET_SURVEY_AVERAGE.
  ENDIF.

  CLEAR LT_EXCL.

  IF GV_MENU_EXISTS IS INITIAL. " MEAL 버튼
    APPEND 'MEAL' TO LT_EXCL.
  ENDIF.

  IF GV_SURVEY_EXISTS = ABAP_TRUE. " SURVEY 버튼 (둘 중 하나만 노출)
    APPEND 'SURV_C' TO LT_EXCL. "작성 숨김
  ELSE.
    APPEND 'SURV_D' TO LT_EXCL. "내용 숨김
  ENDIF.

  SET PF-STATUS 'S100' EXCLUDING LT_EXCL.
ENDMODULE.
```

## 설명(로직 흐름)

1. 탭 유지
- `TABSTRIP-ACTIVETAB = GV_ACTTAB` 로 마지막 탭을 유지한다.
1. 예약키가 있는 경우에만 “조건 계산”
- 예약키가 초기면 존재여부 체크할 의미가 없어서 실행하지 않는다.
1. 버튼 숨김(EXCLUDING)
- 메뉴가 없으면 `MEAL` 숨김
- 설문이 있으면 “작성(SURV_C)” 숨김 → “내용(SURV_D)”만 보이게
- 설문이 없으면 “내용(SURV_D)” 숨김 → “작성(SURV_C)”만 보이게

## LEVEL 2. Screen 0100의PAI(USER_COMMAND_0100) — 버튼 클릭 시 기능 분기

## 코드: USER_COMMAND_0100 전체

```abap
MODULE USER_COMMAND_0100 INPUT.
  CASE OK_CODE.
    WHEN 'TAB1' OR 'TAB2' OR 'TAB3' OR 'TAB4'.
      GV_ACTTAB = OK_CODE.

    WHEN 'SEARCH'.
      PERFORM GET_BOOKING_INFO.
      PERFORM CHECK_MENU_EXISTS   CHANGING GV_MENU_EXISTS.
      PERFORM CHECK_SURVEY_EXISTS CHANGING GV_SURVEY_EXISTS.
      PERFORM SET_SURVEY_AVERAGE.

      GV_ACTTAB = 'TAB1'. "조회 후 Booking Info.

    WHEN 'BACK' OR 'EXIT' OR 'CANC'.
      LEAVE TO SCREEN 0.

    WHEN 'MEAL'.
      "PERFORM SHOW_MEAL_POPUP.
      PERFORM GET_MEAL_POPUP_DATA.
      CALL SCREEN 0300 STARTING AT 20 5 ENDING AT 85 8.

    WHEN 'SURV_C'.
      GV_SRV_MODE = 'C'.
      CALL SCREEN 0200.

    WHEN 'SURV_D'.
      GV_SRV_MODE = 'D'.
      CALL SCREEN 0200.
  ENDCASE.
  CLEAR OK_CODE.
ENDMODULE.

```

## 설명(핵심 포인트)

- `SEARCH`를 누르면:
    
    예약조회 → 메뉴 존재 체크 → 설문 존재 체크 → 평균 계산까지 한 번에 준비한다.
    
    그리고 탭을 TAB1로 돌려준다.
    
- `MEAL`을 누르면: Screen 0300 팝업으로 이동(기내식 신청 내역 표시)
- `SURV_C/SURV_D`: Screen 0200으로 이동하기 전에 `GV_SRV_MODE`로 “작성/조회” 모드를 먼저 세팅한다.

## LEVEL 3. 예약 조회 + 탭 데이터 채우기(BOOKING/CONNECTION/FLIGHT/CUSTOMER)

이 레벨은 “SEARCH” 눌렀을 때 호출되는 핵심 비즈니스 로직이다.

여기서 Screen 0100 탭1~4에 표시할 데이터가 완성된다.

## 3-1) 예약 조회: GET_BOOKING_INFO 전체

```abap
FORM GET_BOOKING_INFO.

  IF SBOOK-CARRID IS INITIAL OR
     SBOOK-CONNID IS INITIAL OR
     SBOOK-FLDATE IS INITIAL OR
     SBOOK-BOOKID IS INITIAL.
    MESSAGE 'CARRID/CONNID/FLDATE/BOOKID 입력 필수!' TYPE 'E'.
  ENDIF.

  "TAB1 booking 조회
  CLEAR: GS_BOOKING, ZST_BOOKING_G01.

  CALL FUNCTION 'Z_GET_BOOK_DATA_G01'
    EXPORTING
      IV_CARRID  = SBOOK-CARRID
      IV_CONNID  = SBOOK-CONNID
      IV_FLDATE  = SBOOK-FLDATE
      IV_BOOKID  = SBOOK-BOOKID
    IMPORTING
      ES_BOOKING = GS_BOOKING.

  IF GS_BOOKING-CARRID IS INITIAL.
    MESSAGE '해당 예약 데이터 없음' TYPE 'I'.
    EXIT.
  ENDIF.

  ZST_BOOKING_G01 = GS_BOOKING.

  PERFORM SET_RADIO_BUTTONS.

  PERFORM GET_FLIGHT_CONNECTION.  "TAB2 채우기
  PERFORM GET_FLIGHT_INFO.        "TAB3 채우기
  PERFORM GET_CUSTOMER_INFO.      "TAB4 채우기

ENDFORM.
```

### 설명

- 입력 4개가 비어있으면 E로 막는다(필수조건).
- FM 결과를 `ZST_BOOKING_G01`에 대입하면, Screen 0100 탭1이 바로 표시된다.
- 탭2~4는 각각 SPFLI/SFLIGHT/SCUSTOM에서 채운다.

## 3-3) 탭2: 항공 연결정보 GET_FLIGHT_CONNECTION 전체

```abap
FORM GET_FLIGHT_CONNECTION.

  CLEAR ZST_CONN_G01.

  SELECT SINGLE
         CARRID
         CONNID
         FLTIME
         DISTANCE
         DISTID
         FLTYPE
         COUNTRYFR
         CITYFROM
         AIRPFROM
         DEPTIME
         COUNTRYTO
         CITYTO
         AIRPTO
         ARRTIME
    INTO CORRESPONDING FIELDS OF ZST_CONN_G01
    FROM SPFLI
   WHERE CARRID = ZST_BOOKING_G01-CARRID
     AND CONNID = ZST_BOOKING_G01-CONNID.

  IF SY-SUBRC <> 0.
    RETURN.
  ENDIF.

  CLEAR: ZST_CONN_G01-RB_FL_S,
         ZST_CONN_G01-RB_FL_C.

  CASE ZST_CONN_G01-FLTYPE.
    WHEN ''.      "Scheduled
      ZST_CONN_G01-RB_FL_S = 'X'.
    WHEN 'X'.        "Charter
      ZST_CONN_G01-RB_FL_C = 'X'.
  ENDCASE.

  " 국가명(T005T) - 로그인 언어.
  SELECT SINGLE LANDX
    INTO ZST_CONN_G01-COUNTRYFR_T
    FROM T005T
   WHERE LAND1 = ZST_CONN_G01-COUNTRYFR
     AND SPRAS = SY-LANGU.

  SELECT SINGLE LANDX
    INTO ZST_CONN_G01-COUNTRYTO_T
    FROM T005T
   WHERE LAND1 = ZST_CONN_G01-COUNTRYTO
     AND SPRAS = SY-LANGU.

  "공항명(SAIRPORT)
  SELECT SINGLE NAME
    INTO ZST_CONN_G01-AIRPFROM_T
    FROM SAIRPORT
   WHERE ID = ZST_CONN_G01-AIRPFROM.

  SELECT SINGLE NAME
    INTO ZST_CONN_G01-AIRPTO_T
    FROM SAIRPORT
   WHERE ID = ZST_CONN_G01-AIRPTO.

ENDFORM.
```

### 설명

- SPFLI에서 연결 기본 정보
- FLTYPE 라디오버튼 세팅
- 국가명(T005T), 공항명(SAIRPORT) 텍스트까지 붙여서 화면 표시

## 3-4) 탭3: 항공편 정보 GET_FLIGHT_INFO 전체

```abap
FORM GET_FLIGHT_INFO.

  CLEAR ZST_FLINFO_G01.

  "SFLIGHT에서 기본 정보
  SELECT SINGLE
         CARRID
         CONNID
         FLDATE
         PRICE
         CURRENCY
         PLANETYPE
         SEATSMAX
         SEATSOCC
         SEATSMAX_B
         SEATSOCC_B
         SEATSMAX_F
         SEATSOCC_F
    INTO CORRESPONDING FIELDS OF ZST_FLINFO_G01
    FROM SFLIGHT
   WHERE CARRID = ZST_BOOKING_G01-CARRID
     AND CONNID = ZST_BOOKING_G01-CONNID
     AND FLDATE = ZST_BOOKING_G01-FLDATE.

  IF SY-SUBRC <> 0.
    RETURN.
  ENDIF.

  "PAYMENTSUM = 해당 항공편의 예약금액 합계 (SBOOK-FORCURAM)
  SELECT SUM( FORCURAM )
    INTO ZST_FLINFO_G01-PAYMENTSUM
    FROM SBOOK
   WHERE CARRID = ZST_BOOKING_G01-CARRID
     AND CONNID = ZST_BOOKING_G01-CONNID
     AND FLDATE = ZST_BOOKING_G01-FLDATE.

  IF ZST_FLINFO_G01-PAYMENTSUM IS INITIAL.
    ZST_FLINFO_G01-PAYMENTSUM = 0.
  ENDIF.

  "좌석 합계/잔여/예약률 계산
  ZST_FLINFO_G01-TOTAL_MAX =
      ZST_FLINFO_G01-SEATSMAX
    + ZST_FLINFO_G01-SEATSMAX_B
    + ZST_FLINFO_G01-SEATSMAX_F.

  ZST_FLINFO_G01-TOTAL_OCC =
      ZST_FLINFO_G01-SEATSOCC
    + ZST_FLINFO_G01-SEATSOCC_B
    + ZST_FLINFO_G01-SEATSOCC_F.

  ZST_FLINFO_G01-FREE_SEAT = ZST_FLINFO_G01-TOTAL_MAX - ZST_FLINFO_G01-TOTAL_OCC.

  IF ZST_FLINFO_G01-TOTAL_MAX > 0.
    ZST_FLINFO_G01-BOOK_RATE =
      ( ZST_FLINFO_G01-TOTAL_OCC * 100 ) / ZST_FLINFO_G01-TOTAL_MAX.
  ELSE.
    CLEAR ZST_FLINFO_G01-BOOK_RATE.
  ENDIF.

ENDFORM.
```

### 설명

- SFLIGHT에서 항공편 기본정보
- SBOOK에서 결제합계(PAYMENTSUM)
- 좌석 총합/점유/잔여/예약률 계산까지 해서 탭3에 표시한다.

## LEVEL 4. Meal(기내식) 처리 흐름 — 존재 체크 → 팝업 데이터 구성 → Screen 0300 표시

이 레벨은 MEAL 버튼이 보이고, 눌렀을 때 팝업이 뜨는 로직이다.

## 4-1) 메뉴 신청 존재 체크: CHECK_MENU_EXISTS 전체

```abap
FORM CHECK_MENU_EXISTS CHANGING CV_EXISTS TYPE ABAP_BOOL.
  DATA LS_REQ TYPE ZTMENUREQ_F01.

  CLEAR CV_EXISTS.

  SELECT SINGLE * INTO @LS_REQ
    FROM ZTMENUREQ_F01
   WHERE CARRID = @ZST_BOOKING_G01-CARRID
     AND CONNID = @ZST_BOOKING_G01-CONNID
     AND FLDATE = @ZST_BOOKING_G01-FLDATE
     AND BOOKID = @ZST_BOOKING_G01-BOOKID.

  IF SY-SUBRC = 0.
    CV_EXISTS = ABAP_TRUE.
  ENDIF.
ENDFORM.

```

### 설명

- 예약 키로 기내식 신청 테이블(ZTMENUREQ_F01) 조회
- 있으면 CV_EXISTS = X
- 이 값이 Screen 0100 PBO에서 MEAL 버튼 숨김/표시를 결정한다.

## 4-2) 팝업 표시 데이터 구성: GET_MEAL_POPUP_DATA 전체

```abap
FORM GET_MEAL_POPUP_DATA.

  DATA: LS_REQ TYPE ZTMENUREQ_F01.

  "예약 선행 체크
  IF ZST_BOOKING_G01-CARRID IS INITIAL
  OR ZST_BOOKING_G01-CONNID IS INITIAL
  OR ZST_BOOKING_G01-FLDATE IS INITIAL
  OR ZST_BOOKING_G01-BOOKID IS INITIAL.
    RETURN.
  ENDIF.

  "1) 기내식 신청내역
  SELECT SINGLE *
    INTO @LS_REQ
    FROM ZTMENUREQ_F01
   WHERE CARRID = @ZST_BOOKING_G01-CARRID
     AND CONNID = @ZST_BOOKING_G01-CONNID
     AND FLDATE = @ZST_BOOKING_G01-FLDATE
     AND BOOKID = @ZST_BOOKING_G01-BOOKID.

  IF SY-SUBRC <> 0.
    RETURN.
  ENDIF.

  GS_MENU_POP-MENUNUMBER = LS_REQ-MENUNUMBER.

  "2) 메뉴 구성
  SELECT SINGLE
         STARTER,
         MAINCOURSE,
         DESSERT
    INTO ( @GS_MENU_POP-APP_ID,
           @GS_MENU_POP-MAIN_ID,
           @GS_MENU_POP-DES_ID )
    FROM SMENU
   WHERE CARRID     = @ZST_BOOKING_G01-CARRID
     AND MENUNUMBER = @GS_MENU_POP-MENUNUMBER.

  "3) 텍스트
  PERFORM GET_MEAL_TEXT USING GS_MENU_POP-APP_ID  CHANGING GS_MENU_POP-APP_TXT.
  PERFORM GET_MEAL_TEXT USING GS_MENU_POP-MAIN_ID CHANGING GS_MENU_POP-MAIN_TXT.
  PERFORM GET_MEAL_TEXT USING GS_MENU_POP-DES_ID  CHANGING GS_MENU_POP-DES_TXT.

ENDFORM.
```

### 설명

- 신청내역에서 MENUNUMBER 가져오고
- SMENU에서 starter/main/dessert 번호를 가져온 다음
- SMEALT에서 각각의 텍스트를 읽어 `GS_MENU_POP`에 채운다.
- Screen 0300은 `GS_MENU_POP`를 바인딩해 표시한다.

## 4-3) 메뉴 텍스트: GET_MENU_TEXT / GET_MEAL_TEXT (너 코드에 있는 것 전부 포함)

### GET_MENU_TEXT 전체

```abap
FORM GET_MENU_TEXT USING    IV_MENUNUMBER TYPE S_MENU
                   CHANGING CV_TEXT      TYPE CHAR50.

  DATA LV_MEALNUMBER TYPE SMEALT-MEALNUMBER.
  CLEAR: CV_TEXT, LV_MEALNUMBER.

  "SMENU: 메뉴번호 -> mealnumber
  SELECT SINGLE MENUNUMBER
    FROM SMENU
    INTO @LV_MEALNUMBER
   WHERE MENUNUMBER = @IV_MENUNUMBER.

  IF SY-SUBRC <> 0 OR LV_MEALNUMBER IS INITIAL.
    CV_TEXT = 'Unknown Menu'.
    RETURN.
  ENDIF.

  "SMEALT: mealnumber -> 메뉴명 (로그온 언어)

  IF SY-SUBRC <> 0 OR CV_TEXT IS INITIAL.

      FROM SMEALT
      INTO @CV_TEXT
     WHERE MEALNUMBER = @LV_MEALNUMBER
       AND SPRACHE      = 'E'.
  ENDIF.

ENDFORM.
```

## LEVEL 6. Survey(Screen 0200) — 한 화면에서 작성/조회의 핵심 설계

이 레벨이 프로젝트에서 제일 중요한 부분이야.

Screen 200은 분리 금지 조건 때문에, **모드에 따라 화면을 바꾸는 기술**이 필요하고, 그 역할을 `GV_SRV_MODE`와 `LOOP AT SCREEN`이 한다.

## 6-1) STATUS_0200(PBO) 전체

```abap
MODULE STATUS_0200 OUTPUT.
  DATA LT_EXCL_200 TYPE TABLE OF SY-UCOMM.

  SET TITLEBAR 'T200'.

  "1) 모드에 따라 화면 데이터 준비
  IF GV_SRV_MODE = 'D'.
    PERFORM LOAD_SURVEY_DATA.      "DB -> ZTSURVEY_G01
  ELSE.
    PERFORM INIT_SURVEY_CREATE.    "ZST_BOOKING_G01 -> ZTSURVEY_G01 + SVYDATE
  ENDIF.

  "2) SAVE 버튼 제어 (조회 모드에서는 숨김)
  CLEAR LT_EXCL_200.
  IF GV_SRV_MODE = 'D'.
    APPEND 'SAVE' TO LT_EXCL_200.
  ENDIF.
  SET PF-STATUS 'S200' EXCLUDING LT_EXCL_200.

  "3) 입력 필드 제어
  LOOP AT SCREEN.

    "상단 Booking Info는 항상 수정 불가 (Screen Painter에서 Group1 = 'BKI')
    IF SCREEN-GROUP1 = 'BKI'.
      SCREEN-INPUT = 0.
      MODIFY SCREEN.
      CONTINUE.
    ENDIF.

    "Survey 영역 (Screen Painter에서 Group1 = 'SRV')
    IF SCREEN-GROUP1 = 'SRV'.

      "조회/작성 모드별 입력 제어
      IF GV_SRV_MODE = 'D'.
        SCREEN-INPUT = 0.
      ELSE.
        SCREEN-INPUT = 1.
      ENDIF.

      "Survey Date는 현재일자 표시만(수정 불가) 요구면 유지
      IF SCREEN-NAME = 'ZTSURVEY_G01-SVYDATE'.
        SCREEN-INPUT = 0.
      ENDIF.

      MODIFY SCREEN.
    ENDIF.

  ENDLOOP.

ENDMODULE.
```

### 설명(단계별)

1. 데이터 준비
- Display(D): DB에서 설문을 SELECT 해서 `ZTSURVEY_G01`에 채움
- Create(C): `ZTSURVEY_G01`에 예약정보/고객정보를 복사하고 날짜를 오늘로 채움
    
    (이렇게 해야 200 화면 상단이 채워지고, 하단은 입력 상태가 된다)
    
1. SAVE 버튼 노출
- 조회 모드(D)면 SAVE 숨김
- 작성 모드(C)면 SAVE 표시
1. 입력/수정 제어
- BKI 그룹: 항상 입력 불가(상단 고정)
- SRV 그룹: 모드에 따라 입력 허용/차단
- SVYDATE: 작성 모드에서도 입력 불가(오늘 날짜 표시만)

## 6-2) USER_COMMAND_0200(PAI) 전체

| Menu 신청 | Survey 존재 | 노출 버튼 |
| --- | --- | --- |
| X | X | `SURV_C` |
| X | O | `SURV_D` |
| O | X | `MEAL` + `SURV_C` |
| O | O | `MEAL` + `SURV_D` |

```abap
MODULE USER_COMMAND_0200 INPUT.
  CASE OK_CODE.
    WHEN 'SAVE'.
      IF GV_SRV_MODE = 'D'.
        MESSAGE '조회 모드에서는 저장할 수 없습니다.' TYPE 'I'.
        CLEAR OK_CODE.
        RETURN.
      ENDIF.

      PERFORM VALIDATE_SURVEY.
      PERFORM SAVE_SURVEY.

      MESSAGE 'Survey 저장 완료' TYPE 'S'.

      "저장 후 내용 모드로 전환
      GV_SRV_MODE = 'D'.
      PERFORM CHECK_SURVEY_EXISTS CHANGING GV_SURVEY_EXISTS.
      PERFORM SET_SURVEY_AVERAGE.

    WHEN 'BACK' OR 'EXIT' OR 'CANC'.
      LEAVE TO SCREEN 0.
  ENDCASE.

  CLEAR OK_CODE.
ENDMODULE.
```

### 설명(저장 플로우)

- 작성 모드(C)만 저장 가능
- 저장 순서
    1. `VALIDATE_SURVEY`로 필수값 체크
    2. `SAVE_SURVEY`로 DB 저장
    3. 모드를 D로 바꾸고
    4. 설문 존재여부/평균을 다시 계산하여 메인 화면 반영 준비

## 6-3) 설문 존재 체크: CHECK_SURVEY_EXISTS 전체

```abap
FORM CHECK_SURVEY_EXISTS CHANGING CV_EXISTS TYPE ABAP_BOOL.
  CLEAR: CV_EXISTS, ZTSURVEY_G01.

  SELECT SINGLE * INTO @ZTSURVEY_G01
    FROM ZTSURVEY_G01
   WHERE CARRID = @ZST_BOOKING_G01-CARRID
     AND CONNID = @ZST_BOOKING_G01-CONNID
     AND FLDATE = @ZST_BOOKING_G01-FLDATE
     AND BOOKID = @ZST_BOOKING_G01-BOOKID.

  IF SY-SUBRC = 0.
    CV_EXISTS = ABAP_TRUE.
  ENDIF.
ENDFORM.
```

### 설명

- 설문이 있는지 체크하면서, 있으면 설문 1건을 `ZTSURVEY_G01`에 같이 담는다.
- 그래서 평균 계산은 별도 SELECT 없이 바로 가능해진다.

## 6-4) 평균 계산: SET_SURVEY_AVERAGE 전체

```abap
FORM SET_SURVEY_AVERAGE.
  DATA: LV_SUM TYPE I,
        LV_AVG TYPE P LENGTH 7 DECIMALS 1.

  ZST_BOOKING_G01-SURVEY_AVG = 0.

  IF GV_SURVEY_EXISTS IS INITIAL.
    RETURN.
  ENDIF.

  LV_SUM = ZTSURVEY_G01-EVALU1
         + ZTSURVEY_G01-EVALU2
         + ZTSURVEY_G01-EVALU3
         + ZTSURVEY_G01-EVALU4
         + ZTSURVEY_G01-EVALU5.

  LV_AVG = LV_SUM / 5.

  ZST_BOOKING_G01-SURVEY_AVG = LV_AVG.
ENDFORM.
```

### 설명

- 설문 없으면 평균 0
- 있으면 evalu1~5 합/5로 평균 내서 `ZST_BOOKING_G01-SURVEY_AVG`에 넣는다(메인 화면 표시용)

## 6-5) 설문 로드: LOAD_SURVEY_DATA 전체

```abap
FORM LOAD_SURVEY_DATA.
  CLEAR ZTSURVEY_G01.

  SELECT SINGLE * INTO @ZTSURVEY_G01
    FROM ZTSURVEY_G01
   WHERE CARRID = @ZST_BOOKING_G01-CARRID
     AND CONNID = @ZST_BOOKING_G01-CONNID
     AND FLDATE = @ZST_BOOKING_G01-FLDATE
     AND BOOKID = @ZST_BOOKING_G01-BOOKID.

  IF SY-SUBRC <> 0.
    MESSAGE 'Survey 데이터가 없습니다. 작성 화면으로 전환합니다.' TYPE 'I'.
    GV_SRV_MODE = 'C'.
    PERFORM INIT_SURVEY_CREATE.
  ENDIF.
ENDFORM.
```

### 설명

- 조회 모드(D)로 들어왔는데 DB에 설문이 없다면, 작성 모드(C)로 돌려서 “작성 화면”을 보여준다.

## 6-6) 설문 작성 초기화: INIT_SURVEY_CREATE 전체

```abap
FORM INIT_SURVEY_CREATE.
  CLEAR ZTSURVEY_G01.

  ZTSURVEY_G01-CARRID   = ZST_BOOKING_G01-CARRID.
  ZTSURVEY_G01-CONNID   = ZST_BOOKING_G01-CONNID.
  ZTSURVEY_G01-FLDATE   = ZST_BOOKING_G01-FLDATE.
  ZTSURVEY_G01-BOOKID   = ZST_BOOKING_G01-BOOKID.
  ZTSURVEY_G01-CUSTOMID = ZST_BOOKING_G01-CUSTOMID.
  ZTSURVEY_G01-CUSTNAME = ZST_BOOKING_G01-CUSTNAME.

  ZTSURVEY_G01-SVYDATE = SY-DATUM.
ENDFORM.
```

### 설명

- Screen 0200이 `ZTSURVEY_G01-*`로 바인딩되어 있기 때문에, 작성 모드에서는 상단 표시를 위해 예약/고객 값을 `ZTSURVEY_G01`에 세팅해줘야 한다.
- 설문 날짜는 오늘로 자동 세팅한다.

## 6-7) 필수 체크: VALIDATE_SURVEY 전체

```abap
FORM VALIDATE_SURVEY.
  IF ZTSURVEY_G01-EVALU1 IS INITIAL OR
     ZTSURVEY_G01-EVALU2 IS INITIAL OR
     ZTSURVEY_G01-EVALU3 IS INITIAL OR
     ZTSURVEY_G01-EVALU4 IS INITIAL OR
     ZTSURVEY_G01-EVALU5 IS INITIAL.
    MESSAGE 'Service 1~5는 필수 입력입니다.' TYPE 'E'.
  ENDIF.
ENDFORM.
```

### 설명

- `E` 메시지라서 하나라도 비면 저장 로직으로 못 내려간다.

## 6-8) 저장: SAVE_SURVEY 전체

```abap
FORM SAVE_SURVEY.
  MODIFY ZTSURVEY_G01 FROM @ZTSURVEY_G01.
  IF SY-SUBRC <> 0.
    INSERT ZTSURVEY_G01 FROM @ZTSURVEY_G01.
  ENDIF.
ENDFORM.
```

### 설명

- UPDATE/INSERT(Upsert) 방식
- 저장 후 메인 화면에서 설문 존재 플래그가 X로 바뀌고, 버튼은 “내용”만 보이게 된다
