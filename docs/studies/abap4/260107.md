## cf. ABAP on HANA 등장 배경

### 🔴 기존 방식 (Classical ABAP)

```
[과거] DB → 모든 데이터 → ABAP → 계산
┌───────────────────────────┐
│     Application Layer     │        
│        (ABAP)             │
│                           │
│  -LOOP                   │
│  -READTABLE             │
│  - 계산 로직               │
│  - 집계 / 필터링           │
│                           │
└───────────▲───────────────┘
            │ 모든 데이터
            │ (대량 전송)
┌───────────┴───────────────┐
│Database Layer       │
│        (DB)               │
│                           │
│  - 데이터 저장             │
│  - 단순SELECT            │
│                           │
└───────────────────────────┘
```

### 🟢 현재 방식 (ABAP on HANA / New Programming Approach)

```
[현재] DB(계산 완료) → 결과만 → ABAP

┌───────────────────────────┐         
│     Application Layer     │    
│        (ABAP)             │
│                           │
│  - 결과 사용               │
│  - UI / 흐름 제어          │
│                           │
└───────────▲───────────────┘
            │ 결과만 전송
            │ (최소 데이터)
┌───────────┴───────────────┐
│Database Layer       │
│        (HANA)             │
│                           │
│  - 계산 로직               │
│  - 필터링                  │
│  - 집계                    │
│  -JOIN                    │
│  (CDS /SQL / AMDP)       │
│                           │
└───────────────────────────┘
```

## 0. Bottom-Up Approach 개요

### 정의

- **Database(HANA)에서 먼저 모델링**
- 이후 ABAP에서 이를 **소비(consume)** 하는 방식
- Code Push-down 전략의 실현 방법 중 하나

```
HANA (View/Procedure 생성)
        ↓
ABAP에서 호출
```

## 1. Bottom-Up Approach – 초기 방식 (AS ABAP < 7.4 SP02)

Figure 32 기준: Direct Access (Native SQL)

![image.png](attachment:46e50465-efd5-4f74-97aa-77ff114484c5:image.png)

- 전체 구조

```
AS ABAP
 └─ NativeSQL로 직접 접근
    ├─ SAP HANA Views
    └─ StoredProcedures

SAP HANA
 ├─Column Views
 ├─ StoredProcedures
 └─ Delivery Unit
```

- 핵심 특징

| 항목 | 내용 |
| --- | --- |
| 접근 방식 | Native SQL |
| ABAP 객체 | 없음 (직접 호출) |
| Transport | ABAP CTS와 분리 |
| DB 종속성 | 매우 높음 |
| 재사용성 | 낮음 |

### 문제점

- Native SQL 필수
- DB 독립성 상실
- ABAP Transport와 HANA Transport 이원화
- 유지보수 및 라이프사이클 관리 어려움

### 사용 시점

- ABAP 7.4 SP02 이전
- SAP HANA가 Secondary DB인 경우
- 레거시 시스템

## 2. Bottom-Up Approach – 개선된 방식 (AS ABAP ≥ 7.4 SP02)

Figure 33 기준: Proxy Objects 사용

![image.png](attachment:ebd80cfd-1de6-4c9c-ba71-312ca56ad967:image.png)

### 핵심 개념

> HANA 객체를 ABAP Repository 안의 Proxy 객체로 감싸서 사용
> 

### 전체 구조

```
AS ABAP
 ├─ExternalView
 ├─DatabaseProcedure Proxy
 └─ SAP HANA Transport Container
        ↑
       expose / transport
        ↓
SAP HANA
 ├─ SAP HANA Views
 ├─ StoredProcedures
 └─ Delivery Unit
```

### 새로 도입된 ABAP 객체

| ABAP 객체 | 역할 |
| --- | --- |
| External View | HANA Column View를 ABAP Dictionary에서 표현 |
| Database Procedure Proxy | HANA Stored Procedure의 ABAP Proxy |
| SAP HANA Transport Container | ABAP–HANA Transport 동기화 |

### 장점

| 항목 | 개선 내용 |
| --- | --- |
| SQL 사용 | Native SQL 불필요 |
| DB 독립성 | 향상 |
| Transport | CTS로 통합 관리 |
| 유지보수 | 쉬움 |
| 재사용성 | 높음 |

## 3. Bottom-Up Approach 접근 방식 비교 요약

| 구분 | Direct Access | Proxy Objects |
| --- | --- | --- |
| ABAP 버전 | < 7.4 SP02 | ≥ 7.4 SP02 |
| SQL 방식 | Native SQL | Open SQL |
| ABAP 객체 | 없음 | External View / Proxy |
| Transport | 분리 | 통합 |

## 8. Open SQL, Native SQL, Database Interface, Open SQL의 강화 정리

![image.png](attachment:c059f408-f93d-477f-93cb-1a9e750fdb23:image.png)

![image.png](attachment:4b93b64a-7da9-47bc-8f15-aaed0f519af9:image.png)

```ruby
ABAP Program
   │
   ├─ Open SQL        (DB 독립) " ABAP 전용의 DB 독립적인 SQL 인터페이스
   └─ Native SQL      (DB 종속) " 특정 데이터베이스의 고유 SQL 문법을 직접 사용
        ↓
Database Interface (DBI) " ABAP과 실제 데이터베이스 사이의 중간 계층
        ↓
Database-Specific Library (DBSL)
        ↓
Database (HANA, Oracle, DB2 ...)
```

### Open SQL vs Standard SQL vs Native SQL 핵심 비교 표

| 구분 | Open SQL | Standard SQL | Native SQL |
| --- | --- | --- | --- |
| 사용 위치 | ABAP | DB 일반 | ABAP |
| DB 독립성 | 높음 | 중간 | 없음 |
| SQL 범위 | 제한적 | 광범위 | DB 전체 |
| DDL/DCL | 불가 | 가능 | 가능 |
| 이식성 | 매우 좋음 | 중간 | 매우 나쁨 |

### Open SQL 제한 사항 → 따라서 Open SQL이 강화되기 시작함

| 제한 항목 | 설명 |
| --- | --- |
| No expressions | CASE, 계산식, 함수 사용 불가 |
| Limited JOIN types | INNER JOIN 중심, 제약 많음 |
| No UNION / UNION ALL | 결과 집합 결합 불가 |
| Additional restrictions | 서브쿼리, 조건식 제한 |

### Open SQL 진화 단계

```sql
SQL-92 Standard
      ▲
StandardizedOpenSQL (≥ ABAP7.5)
      ▲
EnhancedOpenSQL (ABAP7.4)
      ▲
OpenSQL (Before ABAP7.4)
```

## 9. New Open SQL Syntax

### 구버전 코드 vs 신버전 코드

```sql
*---------------------------------------------------------------------*
* 변경 1) Host Variable 표기: @ 필수
*---------------------------------------------------------------------*

" [OLD] (대략 ABAP 7.40 이전 스타일)
SELECT * FROM sbook
  INTO TABLE it_bookings
  WHERE customid = lv_customer.     " 변수 표시 없음

" [NEW] (New Open SQL)
SELECT FROM sbook
  FIELDS *
  WHERE customid = @lv_customer     " ABAP 변수는 @로 명확히 표시
  INTO TABLE @it_bookings.          " 타깃도 @ 가능(권장)
```

```sql
*---------------------------------------------------------------------*
* 변경 2) INTO 절 위치: SELECT 끝으로 이동 (ABAP 7.50)
*---------------------------------------------------------------------*

" [OLD] INTO가 FROM 전에 위치
SELECT carrid connid fldate
  INTO TABLE it_bookings
  FROM sbook
  WHERE customid = lv_customer
  UP TO 10 ROWS.

" [NEW] INTO는 SELECT 문 끝
SELECT carrid, connid, fldate
  FROM sbook
  WHERE customid = @lv_customer
  INTO TABLE @it_bookings
  UP TO 10 ROWS.
```

```sql
*---------------------------------------------------------------------*
* 변경 3) FIELDS 키워드 도입 (ABAP 7.50)
*---------------------------------------------------------------------*

" [OLD] SELECT 다음에 바로 필드 나열
SELECT carrid connid fldate
  FROM sbook
  WHERE customid = lv_customer
  INTO TABLE it_bookings.

" [NEW] FROM 먼저 → FIELDS로 필드 목록 명시 (ADT 자동완성 유리)
SELECT FROM sbook
  FIELDS carrid, connid, fldate
  WHERE customid = @lv_customer
  INTO TABLE @it_bookings.
```

```sql
*---------------------------------------------------------------------*
* 변경 4) UNION / UNION ALL 지원 + INTO는 마지막에 1번만
*---------------------------------------------------------------------*

" [OLD] UNION 미지원 → ABAP에서 합치기
SELECT id name FROM table_a INTO TABLE @DATA(it_a).
SELECT id name FROM table_b INTO TABLE @DATA(it_b).
APPEND LINES OF it_b TO it_a.

" [NEW] SQL에서 UNION 가능, INTO는 맨 끝에 1번 
SELECT FROM table_a "SELECT 절에 * 못 옴 ! FIELDS명도 두 테이브 모두 같아야 함
  FIELDS id, name  
UNION ALL
SELECT FROM table_b
  FIELDS id, name
INTO TABLE @DATA(it_union).
```

```sql
*---------------------------------------------------------------------*
* 변경 5) Expressions(계산식) 지원 강화 (Enhanced Open SQL, 7.40 SP05 계열)
*---------------------------------------------------------------------*

" [OLD] DB에서 계산 어려움 → ABAP에서 후처리
SELECT carrid loccuram FROM sbook INTO TABLE @DATA(it_old).
LOOP AT it_old ASSIGNING FIELD-SYMBOL(<r>).
  <r>-loccuram = <r>-loccuram * 1.1.   " ABAP 계산
ENDLOOP.

" [NEW] DB에서 계산식 수행 (Code Push-down)
SELECT FROM sbook
  FIELDS carrid,
         loccuram,
         loccuram * 1.1 AS loccuram_taxed
  INTO TABLE @DATA(it_expr).
```

```sql
*---------------------------------------------------------------------*
* 변경 6) CASE 지원 (DB에서 조건 분기)
*---------------------------------------------------------------------*

" [OLD] CASE를 ABAP에서 처리
SELECT carrid smoker FROM sbook INTO TABLE @DATA(it_old_case).
LOOP AT it_old_case ASSIGNING FIELD-SYMBOL(<c>).
  IF <c>-smoker = 'X'.
    " smoker_text = 'SMOKER'
  ELSE.
    " smoker_text = 'NON-SMOKER'
  ENDIF.
ENDLOOP.

" [NEW] CASE를 Open SQL에서 바로 처리
SELECT FROM sbook
  FIELDS carrid,
         smoker,
         CASE
           WHEN smoker = 'X' THEN 'SMOKER'
           ELSE 'NON-SMOKER'
         END AS smoker_text
  INTO TABLE @DATA(it_case).
```

```sql
*---------------------------------------------------------------------*
* 변경 7) COALESCE 지원 (NULL 대체)
*---------------------------------------------------------------------*

" [OLD] NULL 처리 위해 ABAP에서 IF/IS INITIAL
SELECT carrid order_date fldate FROM sbook INTO TABLE @DATA(it_old_null).
" LOOP 돌며 order_date 비면 fldate로 대체하는 로직 필요

" [NEW] COALESCE로 DB에서 처리
SELECT FROM sbook
  FIELDS carrid,
         COALESCE( order_date, fldate ) AS effective_date
  INTO TABLE @DATA(it_coalesce).
```

```sql
*---------------------------------------------------------------------*
* 변경 8) CAST 지원 (타입 변환)
*---------------------------------------------------------------------*

" [OLD] ABAP에서 타입 변환
SELECT carrid loccuram FROM sbook INTO TABLE @DATA(it_old_cast).
" LOOP + CONV fltp( ... ) 등의 ABAP 변환 필요

" [NEW] DB에서 CAST로 변환
SELECT FROM sbook
  FIELDS carrid,
         CAST( loccuram AS FLTP ) AS amount_flt
  INTO TABLE @DATA(it_cast).
```

```sql
*---------------------------------------------------------------------*
* 변경 9) 수학 함수 지원 (DIV, MOD, ABS, FLOOR, CEIL 등)
*---------------------------------------------------------------------*

" [OLD] ABAP에서 계산
SELECT carrid loccuram FROM sbook INTO TABLE @DATA(it_old_math).
" LOOP에서 abs, floor/ceil 등을 ABAP로 구현/호출

" [NEW] DB에서 함수 수행
SELECT FROM sbook
  FIELDS carrid,
         ABS( loccuram )    AS abs_amount,
         FLOOR( loccuram )  AS floor_amt,
         CEIL( loccuram )   AS ceil_amt
  INTO TABLE @DATA(it_math).
```

```sql
*---------------------------------------------------------------------*
* 변경 10) 문자열 결합 && 및 문자열 함수 지원
*---------------------------------------------------------------------*

" [OLD] ABAP에서 CONCATENATE
SELECT id name_first name_last FROM scustom INTO TABLE @DATA(it_old_str).
" LOOP에서 CONCATENATE name_first name_last INTO full_name SEPARATED BY space.

" [NEW] DB에서 문자열 결합 (&&)
SELECT FROM scustom
  FIELDS id,
         name_first && ' ' && name_last AS full_name
  INTO TABLE @DATA(it_concat).

" [NEW] 문자열 함수 예시
SELECT FROM scustom
  FIELDS id,
         UPPER( name_last )  AS last_upper,
         LOWER( name_first ) AS first_lower,
         LENGTH( name_last ) AS last_len
  INTO TABLE @DATA(it_str_func).
```

```sql
*---------------------------------------------------------------------*
* 변경 11) ORDER BY / UP TO / OFFSET를 SELECT 끝쪽에서 조합 (페이징)
*---------------------------------------------------------------------*

" [OLD] 페이징/정렬 조합이 제한적이거나 후처리 필요
SELECT carrid connid fldate FROM sbook INTO TABLE @DATA(it_old_order).

" [NEW] DB에서 정렬 + 제한 + 오프셋 처리
SELECT FROM sbook
  FIELDS carrid, connid, fldate
  ORDER BY fldate DESC
  INTO TABLE @DATA(it_page)
  UP TO 20 ROWS
  OFFSET 10.
```
